# [다국어] 화면 요구사항

---

## LABEL 관리

**화면 목적**: 다국어 라벨을 등록하고 언어별 텍스트를 관리하는 화면. 시스템 UI에 표시되는 텍스트(버튼명, 컬럼명, 메시지 등)의 다국어 번역을 중앙에서 관리한다.

**접근 경로**: GNB > 다국어/가이드 > LABEL 관리

**화면 유형**: 표준 CRUD 화면 (템플릿 A 참조)

### 화면 레이아웃 구성

| 영역 | 설명 |
|------|------|
| 검색 필터 영역 | 상단, 검색 유형 + 검색어 + 언어 필터 + 라벨 유형 필터 + 조회 건수 + 조회 버튼 |
| 도구 모음 | 목록 상단 우측, 엑셀 / 출력 버튼 |
| 목록 영역 | 중앙, LABEL 목록 테이블 |
| 페이지 네비게이션 | 목록 하단 |
| 버튼 영역 | 목록 하단, 등록 버튼 |

### 검색 / 필터

| 필터 항목명 | 입력 유형 | 필수 여부 | 기본값 | 유효성 조건 |
|-------------|-----------|-----------|--------|-------------|
| 검색 유형 | 드롭다운 (라벨 ID / 라벨 텍스트) | 선택 | 라벨 ID | - |
| 검색어 | 텍스트 입력 | 선택 | 빈 값 | - |
| 언어 | 드롭다운 (전체 + 언어 목록: ko/en/ja/zh 등) | 선택 | 전체 | API에서 지원 언어 동적 로드 |
| 라벨 유형 | 드롭다운 (전체/버튼/컬럼명/메시지/메뉴/기타) | 선택 | 전체 | - |
| 번역 누락 필터 | 드롭다운 (전체 + 언어 목록: ko/en/ja/zh 등) | 선택 | 전체 | 선택한 언어의 번역이 누락된 항목만 조회 |

### 목록 표시

| 컬럼명 | 데이터 유형 | 정렬 가능 | 비고 |
|--------|-------------|-----------|------|
| 라벨 ID | 텍스트 | 가능 | - |
| 라벨 유형 | 텍스트 | 가능 | 버튼/컬럼명/메시지/메뉴/기타 |
| 한국어 (ko) | 텍스트 | 가능 | 기본 언어 |
| 영어 (en) | 텍스트 | 가능 | - |
| 번역 완료율 | 텍스트 | 가능 | "N/M 언어" 형식 (예: "3/4 언어") |
| 최종 수정일시 | 날짜/시간 | 가능 | - |

- 목록 클릭 시 동작: LABEL 수정 팝업 열림
- 기본 정렬 기준: 라벨 ID 오름차순
- 페이지당 기본 표시 건수: 20건
- 번역이 누락된 언어가 있는 행은 시각적으로 강조 표시 (노란 배경)

### 등록 / 수정 팝업

| 항목명 | 입력 유형 | 필수 여부 | 수정 가능 여부 | 유효성 조건 |
|--------|-----------|-----------|----------------|-------------|
| 라벨 ID | 텍스트 입력 | 필수 | 등록 시만 가능 | 영문/숫자/점/밑줄, 최대 100자, 중복 불가 |
| 라벨 유형 | 드롭다운 (버튼/컬럼명/메시지/메뉴/기타) | 필수 | 가능 | - |
| 설명 | 텍스트 입력 | 선택 | 가능 | 최대 200자, 라벨의 사용 맥락 설명 |
| 한국어 (ko) | 텍스트 입력 | 필수 | 가능 | 최대 200자 |
| 영어 (en) | 텍스트 입력 | 선택 | 가능 | 최대 200자 |
| 일본어 (ja) | 텍스트 입력 | 선택 | 가능 | 최대 200자 |
| 중국어 (zh) | 텍스트 입력 | 선택 | 가능 | 최대 200자 |

- 지원 언어가 추가되면 언어별 입력 필드가 동적으로 추가되어야 한다.
- 각 언어 필드 옆에 번역 상태 아이콘 표시 (입력됨=초록 체크, 미입력=회색 빈칸)

### 버튼 및 액션

| 버튼명 | 노출 조건 | 클릭 동작 | 권한 제어 |
|--------|-----------|-----------|-----------|
| 등록 | 항상 노출 | LABEL 등록 팝업 열기 | 쓰기 권한 |
| 엑셀 | 항상 노출 | 검색 결과 엑셀 다운로드 (모든 언어 포함) | 전체 |
| 출력 | 항상 노출 | 목록 인쇄 | 전체 |
| 엑셀 업로드 | 항상 노출 | 엑셀 파일에서 다국어 라벨을 일괄 등록/수정 | 쓰기 권한 |
| 저장 (팝업) | 팝업 열림 시 | LABEL 정보 저장 | 쓰기 권한 |
| 삭제 (팝업) | 수정 모드 시 | 확인 팝업 후 삭제 | 쓰기 권한 |

### 개별 기능 요구사항

- 엑셀 업로드로 다국어 라벨을 일괄 등록/수정할 수 있어야 한다 (라벨ID + 각 언어별 텍스트)
- 엑셀 다운로드 시 모든 지원 언어의 텍스트가 컬럼으로 포함되어야 한다
- 번역 누락 필터: 특정 언어의 번역이 누락된 라벨만 필터링하여 조회 가능
- 라벨 ID는 네이밍 규칙을 따라야 한다 (예: btn.save, col.userId, msg.success.save)

### 비기능 요구사항

| 항목 | 요구사항 |
|------|----------|
| 응답 속도 | 목록 조회 결과가 3초 이내에 표시되어야 한다 |

## 다국어 관리

**화면 목적**: 사이트 구분별로 UI 라벨 ID와 국문/영문 텍스트를 일괄 등록하고 관리하는 화면이다. LABEL 관리보다 더 세밀한 다국어 편집이 가능하다.

**접근 경로**: GNB > 다국어/가이드 > 다국어 관리

**화면 유형**: 표준 CRUD 화면 (인라인 편집 그리드, 템플릿 A 참조)

### 화면 레이아웃 구성

| 영역 | 설명 |
|------|------|
| 검색 필터 영역 | SITE 구분 드롭다운 + LABEL ID + 검색 유형(국문/영문) + 검색어 + 확장 필터(LABEL 구분) + 조회 건수 + 조회 버튼 |
| 목록 영역 | 다국어 라벨 목록 (인라인 편집 가능) |
| 페이지 네비게이션 | 목록 하단 |
| 버튼 영역 | 행 추가 / 선택행 삭제 / 변경사항 저장 / 다수건 등록 |

### 검색 / 필터

| 필터 항목명 | 입력 유형 | 필수 여부 | 기본값 | 유효성 조건 |
|-------------|-----------|-----------|--------|-------------|
| SITE 구분 | 드롭다운 (전체 + SITE 목록) | 선택 | 전체 | - |
| LABEL ID | 텍스트 입력 | 선택 | 빈 값 | - |
| 검색 유형 | 드롭다운 (국문 / 영문) | 선택 | 국문 | - |
| 검색어 | 텍스트 입력 | 선택 | 빈 값 | - |
| LABEL 구분 | 드롭다운 (전체 + 구분 목록) | 선택 (확장) | 전체 | - |

### 목록 표시

| 컬럼명 | 데이터 유형 | 정렬 가능 | 비고 |
|--------|-------------|-----------|------|
| 처리결과 | 텍스트 | 불가 | 저장 시 성공/실패 표시 |
| SITE 구분 | 드롭다운 (편집 가능) | 가능 | - |
| LABEL ID | 텍스트 (편집 가능) | 가능 | - |
| LABEL 구분 | 드롭다운 (편집 가능) | 가능 | - |
| 국문 | 텍스트 (편집 가능) | 가능 | - |
| 영문 | 텍스트 (편집 가능) | 가능 | - |

- 기본 정렬 기준: SITE 구분 > LABEL ID 오름차순
- 페이지당 기본 표시 건수: 20건

### 버튼 및 액션

| 버튼명 | 노출 조건 | 클릭 동작 | 권한 제어 |
|--------|-----------|-----------|-----------|
| 행 추가 | 항상 노출 | 빈 행 추가 | 쓰기 권한 |
| 선택행 삭제 | 행 선택 시 | 선택 행 삭제 | 쓰기 권한 |
| 변경사항 저장 | 항상 노출 | 편집 내용 일괄 저장 | 쓰기 권한 |
| 다수건 등록 | 항상 노출 | 엑셀 업로드 등 대량 등록 팝업 | 쓰기 권한 |

### 개별 기능 요구사항

- SITE 구분과 LABEL 구분을 목록 그리드에서 드롭다운으로 직접 편집할 수 있어야 한다.
- 다수건 등록으로 여러 라벨을 한 번에 업로드할 수 있어야 한다.
- 저장 결과(성공/실패)가 처리결과 컬럼에 행별로 표시되어야 한다.

### 비기능 요구사항

| 항목 | 요구사항 |
|------|----------|
| 응답 속도 | 목록 조회가 3초 이내에 완료되어야 한다 |

## 페이지별 가이드/팁 관리

**화면 목적**: 시스템의 각 페이지 URL에 표시할 가이드 및 팁 정보를 등록하고 관리하는 화면이다. 등록된 가이드는 해당 페이지에서 사용자가 참고할 수 있다.

**접근 경로**: GNB > 다국어/가이드 > 페이지별 가이드/팁 관리

**화면 유형**: 표준 CRUD 화면 (템플릿 A 참조)

### 화면 레이아웃 구성

| 영역 | 설명 |
|------|------|
| 검색 필터 영역 | 상단, PAGE URI + 가이드 설명 + 조회 건수 + 조회 버튼 |
| 도구 모음 | RELOAD ALL 버튼 |
| 목록 영역 | 가이드/팁 목록 테이블 |
| 페이지 네비게이션 | 목록 하단 |
| 버튼 영역 | 등록 버튼 |

### 검색 / 필터

| 필터 항목명 | 입력 유형 | 필수 여부 | 기본값 | 유효성 조건 |
|-------------|-----------|-----------|--------|-------------|
| PAGE URI | 텍스트 입력 | 선택 | 빈 값 | - |
| 가이드 설명 | 텍스트 입력 | 선택 | 빈 값 | - |

### 목록 표시

| 컬럼명 | 데이터 유형 | 정렬 가능 | 비고 |
|--------|-------------|-----------|------|
| PAGE URI | 텍스트 | 가능 | - |
| URI Full Path | 텍스트 | 가능 | 완전한 URL 경로 |
| 가이드 설명 | 텍스트 | 가능 | 말줄임 처리 |
| Reload | 버튼 | 불가 | 해당 페이지 가이드 즉시 재로드 (쓰기 권한) |

- 목록 클릭 시 동작: 가이드/팁 수정 팝업 열림
- 기본 정렬 기준: PAGE URI 오름차순
- 페이지당 기본 표시 건수: 20건

### 등록 / 수정 팝업

| 항목명 | 입력 유형 | 필수 여부 | 수정 가능 여부 | 유효성 조건 |
|--------|-----------|-----------|----------------|-------------|
| PAGE URI | 텍스트 입력 | 필수 | 등록 시만 가능 | 최대 200자 |
| URI Full Path | 텍스트 입력 | 선택 | 가능 | 최대 500자 |
| 가이드 내용 | 텍스트 영역 | 필수 | 가능 | HTML/텍스트 지원, 최대 5000자 |

### 버튼 및 액션

| 버튼명 | 노출 조건 | 클릭 동작 | 권한 제어 |
|--------|-----------|-----------|-----------|
| 등록 | 항상 노출 | 가이드 등록 팝업 열기 | 쓰기 권한 |
| RELOAD ALL | 항상 노출 | 전체 페이지 가이드 정보 일괄 재로드 | 쓰기 권한 |
| Reload (목록) | 목록 행 내 | 해당 페이지의 가이드 정보 개별 재로드 | 쓰기 권한 |
| 저장 (팝업) | 팝업 열림 시 | 가이드 정보 저장 | 쓰기 권한 |
| 삭제 (팝업) | 수정 모드 시 | 확인 팝업 후 삭제 | 쓰기 권한 |

### 개별 기능 요구사항

- 목록에서 개별 Reload 버튼으로 해당 페이지의 가이드 정보만 즉시 갱신할 수 있어야 한다.
- RELOAD ALL 버튼으로 모든 페이지의 가이드 정보를 한 번에 갱신할 수 있어야 한다.

### 비기능 요구사항

| 항목 | 요구사항 |
|------|----------|
| 응답 속도 | 목록 조회가 3초 이내에 완료되어야 한다 |

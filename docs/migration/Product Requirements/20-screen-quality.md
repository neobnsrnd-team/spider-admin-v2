# [품질 관리] 화면 요구사항

---

## 테스트 시나리오

**화면 목적**: 테스트 수행의 기준이 되는 시나리오를 등록하고 관리하는 화면이다. 시나리오는 테스트케이스를 묶는 상위 단위로, 시나리오 단위로 진척을 관리할 수 있다.

**접근 경로**: GNB > 품질 관리 > 테스트 시나리오

**화면 ID**: FWKT1001

**화면 유형**: 표준 CRUD 화면 (템플릿 A 참조)

### 화면 레이아웃 구성

| 영역 | 설명 |
|------|------|
| 검색 필터 영역 | 상단, 시나리오명/ID 검색 유형 드롭다운 + 검색어 + 사용 여부 드롭다운 + 조회 건수 + 조회 버튼 |
| 도구 모음 | 목록 상단 우측, 엑셀 / PDF / 출력 버튼 |
| 목록 영역 | 중앙, 테스트 시나리오 목록 테이블 |
| 페이지 네비게이션 | 목록 하단 |
| 버튼 영역 | 목록 하단, 등록 버튼 |

### 검색 / 필터

| 필터 항목명 | 입력 유형 | 필수 여부 | 기본값 | 유효성 조건 |
|-------------|-----------|-----------|--------|-------------|
| 검색 유형 | 드롭다운 (시나리오명 / 시나리오 ID) | 선택 | 시나리오명 | - |
| 검색어 | 텍스트 입력 | 선택 | 빈 값 | - |
| 사용 여부 | 드롭다운 (전체 / 사용 / 미사용) | 선택 | 전체 | - |

### 목록 표시

| 컬럼명 | 데이터 유형 | 정렬 가능 | 비고 |
|--------|-------------|-----------|------|
| 시나리오 ID | 텍스트 | 가능 | - |
| 시나리오명 | 텍스트 | 가능 | - |
| 설명 | 텍스트 | 가능 | - |
| 케이스 수 | 숫자 | 가능 | 해당 시나리오에 속한 테스트케이스 수 |
| 사용 여부 | 상태 아이콘 | 불가 | 초록/빨강 원형 아이콘 |
| 등록자 | 텍스트 | 가능 | - |
| 등록일시 | 날짜/시간 | 가능 | - |

- 목록 클릭 시 동작: 시나리오 수정 팝업 열림
- 기본 정렬 기준: 시나리오 ID 오름차순
- 페이지당 기본 표시 건수: 20건

### 등록 / 수정 팝업

| 항목명 | 입력 유형 | 필수 여부 | 수정 가능 여부 | 유효성 조건 |
|--------|-----------|-----------|----------------|-------------|
| 시나리오 ID | 텍스트 입력 | 필수 | 등록 시만 가능 | 영문/숫자, 최대 50자, 중복 불가 |
| 시나리오명 | 텍스트 입력 | 필수 | 가능 | 최대 200자 |
| 설명 | 텍스트 영역 | 선택 | 가능 | 최대 1000자 |
| 사용 여부 | 드롭다운 (Y/N) | 필수 | 가능 | 기본값: Y |

### 버튼 및 액션

| 버튼명 | 노출 조건 | 클릭 동작 | 권한 제어 |
|--------|-----------|-----------|-----------|
| 등록 | 항상 노출 | 시나리오 등록 팝업 열기 | 쓰기 권한 |
| 엑셀 | 항상 노출 | 검색 결과 엑셀 다운로드 | 전체 |
| PDF | 항상 노출 | 검색 결과 PDF 다운로드 | 전체 |
| 출력 | 항상 노출 | 목록 인쇄 | 전체 |
| 저장 (팝업) | 팝업 열림 시 | 시나리오 정보 저장 | 쓰기 권한 |
| 삭제 (팝업) | 수정 모드 시 | 확인 팝업 후 시나리오 삭제 | 쓰기 권한 |

### 개별 기능 요구사항

- 시나리오에 테스트케이스가 등록된 경우 삭제 시 경고 메시지가 표시되어야 한다.
- 엑셀 및 PDF 출력 시 시나리오 정보와 하위 테스트케이스 목록이 함께 포함되어야 한다.

### 비기능 요구사항

| 항목 | 요구사항 |
|------|----------|
| 응답 속도 | 시나리오 목록 조회 결과가 3초 이내에 표시되어야 한다 |

**화면 목적**: 테스트케이스를 등록하고, 카테고리/업무/채널 등으로 분류하며, 수행 이력을 관리하는 화면이다.

**접근 경로**: GNB > 품질 관리 > 테스트케이스 관리

**화면 ID**: FWKT1021

**화면 유형**: 표준 CRUD 화면 (템플릿 A 참조)

## 테스트케이스 영향도 평가

**화면 목적**: 시스템 변경(메뉴 URL, Action, BIZ 컴포넌트, 서비스, 거래 ID, SQL 등)이 기존 테스트케이스에 미치는 영향도를 분석하여 회귀 테스트 범위를 결정하는 화면이다 (조회 전용).

**접근 경로**: GNB > 품질 관리 > 테스트케이스 영향도 평가

**화면 ID**: FWKT1040

**화면 유형**: 조회 전용 화면 (템플릿 B 참조)

### 화면 레이아웃 구성

| 영역 | 설명 |
|------|------|
| 검색 필터 영역 | 상단, 조회 기간 (날짜 범위) + 조회 유형 드롭다운 + 조회 유형별 검색어 + 조회 건수 + 조회 버튼 |
| 도구 모음 | 목록 상단 우측, 엑셀 / 출력 버튼 |
| 목록 영역 | 중앙, 영향도 분석 결과 목록 테이블 |
| 그래프 영역 | 목록 우측 또는 하단, 영향도 시각화 차트 |
| 페이지 네비게이션 | 목록 하단 |

### 검색 / 필터

| 필터 항목명 | 입력 유형 | 필수 여부 | 기본값 | 유효성 조건 |
|-------------|-----------|-----------|--------|-------------|
| 조회 기간 | 날짜 범위 | 선택 | 최근 1개월 | 시작일 ≤ 종료일 |
| 조회 유형 | 드롭다운 (메뉴 URL / Action / Biz 컴포넌트 / 서비스 / 거래 ID / SQL) | 필수 | 메뉴 URL | - |
| 검색어 | 텍스트 입력 | 선택 | 빈 값 | 선택한 조회 유형에 해당하는 값 입력 |

### 목록 표시

| 컬럼명 | 데이터 유형 | 정렬 가능 | 비고 |
|--------|-------------|-----------|------|
| 메뉴 URL | 텍스트 | 가능 | 조회 유형이 메뉴 URL인 경우 |
| 메뉴명 | 텍스트 | 가능 | - |
| SITE 구분 | 텍스트 | 가능 | - |
| 서비스 상태 | 상태 배지 | 가능 | 정상/변경/삭제 등 |
| 영향받는 케이스 수 | 숫자 | 가능 | 해당 항목에 연결된 테스트케이스 수 |
| 최근 수행일 | 날짜 | 가능 | - |

- 목록 행 클릭 시 해당 항목과 연결된 테스트케이스 목록이 하단 또는 팝업에 표시되어야 한다.
- 기본 정렬 기준: 영향받는 케이스 수 내림차순
- 페이지당 기본 표시 건수: 20건

### 버튼 및 액션

| 버튼명 | 노출 조건 | 클릭 동작 | 권한 제어 |
|--------|-----------|-----------|-----------|
| 엑셀 | 항상 노출 | 검색 결과 엑셀 다운로드 | 전체 |
| 출력 | 항상 노출 | 목록 인쇄 | 전체 |

### 개별 기능 요구사항

- 조회 유형 선택 시 검색어 입력 필드의 레이블이 해당 유형에 맞게 변경되어야 한다.
- 그래프 영역에서 영향도 분포(영향받는 케이스 수 기준)를 시각화하여 표시하여야 한다.
- 목록 행 클릭 시 해당 항목에 연관된 테스트케이스 목록을 확인할 수 있어야 한다.

### 비기능 요구사항

| 항목 | 요구사항 |
|------|----------|
| 응답 속도 | 영향도 분석 결과가 5초 이내에 표시되어야 한다 |

**접근 경로**: GNB > 품질 관리 > 프로젝트 관리

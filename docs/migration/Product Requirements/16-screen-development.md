# [개발 관리] 화면 요구사항

---

## 서비스 관리

**화면 목적**: 업무 서비스를 등록하고 컴포넌트를 조합하여 흐름을 구성하는 화면이다.

**접근 경로**: GNB > 개발 관리 > 서비스 관리

**화면 유형**: 목록-상세 전환형 화면 (템플릿 E 참조)

### 화면 레이아웃 구성 - 목록 뷰

| 영역 | 설명 |
|------|------|
| 제목 영역 | 화면 제목 "서비스 관리" + 엑셀 / 출력 / 새로고침 버튼 |
| 검색 필터 영역 | 상단, 검색 유형 드롭다운 + 검색어 + 업무분류 드롭다운 + 조회 건수 + 조회 버튼 |
| 목록 영역 | 중앙, 서비스 목록 테이블 |
| 페이지 네비게이션 | 목록 하단 |
| 버튼 영역 | 목록 하단, 등록 버튼 |

### 화면 레이아웃 구성 - 상세 뷰

| 영역 | 설명 |
|------|------|
| 제목 영역 | "등록" 또는 "수정" 제목 |
| 기본 정보 폼 | 2열 격자 배열의 서비스 기본 정보 입력 폼 |
| 컴포넌트 흐름 영역 | 서비스에 포함된 컴포넌트를 실행 순서대로 표시하고 드래그앤드롭으로 순서 변경이 가능한 영역 |
| 접근 사용자 관리 영역 | 접기/펼치기 가능한 접근 사용자 목록 관리 영역 |
| 버튼 영역 | 저장, 삭제(수정 시), 복제, 목록으로 돌아가기 버튼 |

### 검색 / 필터 (목록 뷰)

| 필터 항목명 | 입력 유형 | 필수 여부 | 기본값 | 유효성 조건 |
|-------------|-----------|-----------|--------|-------------|
| 검색 유형 | 드롭다운 (서비스명 / 서비스 ID) | 선택 | 서비스명 | - |
| 검색어 | 텍스트 입력 | 선택 | 빈 값 | - |
| 업무분류 | 드롭다운 (전체 + 업무분류 목록) | 선택 | 전체 | API에서 동적 로드 |

### 목록 표시 (목록 뷰)

| 컬럼명 | 데이터 유형 | 정렬 가능 | 비고 |
|--------|-------------|-----------|------|
| 서비스 ID | 텍스트 | 가능 | - |
| 서비스명 | 텍스트 | 가능 | - |
| 업무분류 | 텍스트 | 가능 | 업무분류 테이블에서 조인 |
| 서비스 유형 | 텍스트 | 가능 | - |
| 컴포넌트 수 | 숫자 | 가능 | 서비스에 포함된 컴포넌트 건수 |
| 사용 여부 | 상태 아이콘 | 가능 | Y/N, 초록/빨강 원형 아이콘 |
| 최종 수정일시 | 날짜/시간 | 가능 | - |

- 목록 클릭 시 동작: 상세 뷰(수정 모드)로 전환
- 기본 정렬 기준: 서비스 ID 오름차순
- 페이지당 기본 표시 건수: 20건

### 상세 뷰 - 기본 정보

| 항목명 | 입력 유형 | 필수 여부 | 수정 가능 여부 | 유효성 조건 |
|--------|-----------|-----------|----------------|-------------|
| 서비스 ID | 텍스트 입력 | 필수 | 등록 시만 가능 | 영문/숫자, 최대 50자 |
| 서비스명 | 텍스트 입력 | 필수 | 가능 | 최대 100자 |
| 서비스 유형 | 드롭다운 | 필수 | 가능 | 서비스 유형 목록에서 선택 |
| 업무분류 | 드롭다운 (전체 + 업무분류 목록) | 선택 | 가능 | API에서 동적 로드 |
| 설명 | 텍스트 영역 | 선택 | 가능 | 최대 500자 |
| 사용 여부 | 드롭다운 (Y/N) | 필수 | 가능 | 기본값: Y |

### 상세 뷰 - 컴포넌트 흐름 영역

| 항목명 | 데이터 유형 | 비고 |
|--------|-------------|------|
| 컴포넌트 ID | 텍스트 (읽기 전용) | 컴포넌트 검색 팝업에서 선택하여 추가 |
| 컴포넌트명 | 텍스트 (읽기 전용) | - |
| 유형 | 텍스트 (읽기 전용) | 컴포넌트 유형 표시 |
| 파라미터 설정 | 버튼 | 클릭 시 해당 컴포넌트의 파라미터 입력/수정 인라인 영역 확장 |

- 서비스에 포함된 컴포넌트 목록이 실행 순서대로 표시되어야 한다.
- 드래그앤드롭으로 컴포넌트 순서를 변경할 수 있어야 한다.
- 컴포넌트 추가 버튼 클릭 시 컴포넌트 검색 팝업이 열려 컴포넌트를 선택하여 추가할 수 있어야 한다.
- 컴포넌트를 선택한 후 삭제 버튼으로 제거할 수 있어야 한다.
- 파라미터 설정 버튼 클릭 시 해당 컴포넌트의 파라미터를 입력/수정하는 인라인 영역이 확장되어야 한다.
- 인라인 파라미터 영역에는 파라미터 ID, 파라미터명, 값 입력 필드가 표시되어야 한다.

### 상세 뷰 - 접근 사용자 관리 영역 (접기/펼치기)

| 항목명 | 데이터 유형 | 비고 |
|--------|-------------|------|
| 사용자 ID | 텍스트 (읽기 전용) | - |
| 사용자명 | 텍스트 (읽기 전용) | - |

- 이 서비스에 접근 가능한 사용자 목록이 표시되어야 한다.
- 사용자 추가 버튼 클릭 시 사용자 검색 팝업이 열려 사용자를 선택하여 추가할 수 있어야 한다.
- 사용자를 선택한 후 제거 버튼으로 접근 권한을 해제할 수 있어야 한다.
- 접기/펼치기 토글로 영역을 숨기거나 표시할 수 있어야 한다.

### 버튼 및 액션

| 버튼명 | 노출 조건 | 클릭 동작 | 권한 제어 |
|--------|-----------|-----------|-----------|
| 등록 | 목록 뷰에서 항상 노출 | 상세 뷰(등록 모드)로 전환 | 쓰기 권한 |
| 엑셀 | 목록 뷰에서 항상 노출 | 검색 결과 엑셀 다운로드 | 전체 |
| 출력 | 목록 뷰에서 항상 노출 | 목록 인쇄 | 전체 |
| 저장 | 상세 뷰에서 항상 노출 | 서비스 정보 및 컴포넌트 흐름 저장 | 쓰기 권한 |
| 삭제 | 상세 뷰 수정 모드 시 | 확인 팝업 후 서비스 삭제 | 쓰기 권한 |
| 복제 | 상세 뷰 수정 모드 시 | 기존 서비스를 기반으로 새 서비스 ID만 입력받아 복사 등록 | 쓰기 권한 |
| 목록으로 | 상세 뷰에서 항상 노출 | 목록 뷰로 복귀 (이전 검색 조건 유지) | 전체 |
| 컴포넌트 추가 | 상세 뷰 컴포넌트 흐름 영역 | 컴포넌트 검색 팝업 열기 | 쓰기 권한 |
| 컴포넌트 제거 | 상세 뷰 컴포넌트 선택 시 | 선택된 컴포넌트를 흐름에서 제거 | 쓰기 권한 |
| 사용자 추가 | 상세 뷰 접근 사용자 영역 | 사용자 검색 팝업 열기 | 쓰기 권한 |
| 사용자 제거 | 상세 뷰 사용자 선택 시 | 선택된 사용자의 접근 권한 해제 | 쓰기 권한 |

### 개별 기능 요구사항

- 서비스 복제 기능: 기존 서비스의 모든 정보(기본 정보, 컴포넌트 흐름, 접근 사용자)를 기반으로 새 서비스 ID만 입력받아 복사 등록할 수 있어야 한다.
- 복제 버튼 클릭 시 새 서비스 ID를 입력하는 팝업이 표시되어야 하며, 중복 ID 입력 시 "이미 존재하는 서비스 ID입니다" 메시지가 표시되어야 한다.
- 드래그앤드롭으로 컴포넌트를 흐름에 추가하거나 순서를 변경할 수 있어야 한다.
- 컴포넌트 검색 팝업에서는 컴포넌트 ID, 컴포넌트명, 유형으로 검색이 가능하여야 한다.
- 파라미터 설정 인라인 영역은 해당 컴포넌트에 정의된 파라미터 목록을 표시하고 값을 입력할 수 있어야 한다.
- 목록 뷰로 복귀 시 이전 검색 조건과 스크롤 위치가 유지되어야 한다.
- 서비스에 연결된 컴포넌트가 있는 경우 삭제 시 경고 팝업이 표시되어야 한다.

### 비기능 요구사항

| 항목 | 요구사항 |
|------|----------|
| 응답 속도 | 목록 조회 결과가 3초 이내에 표시되어야 한다 |
| 드래그앤드롭 | 컴포넌트 순서 변경 시 화면 반응이 즉시(100ms 이내) 이루어져야 한다 |
| 상세 전환 | 목록 뷰와 상세 뷰 간 전환이 애니메이션과 함께 0.3초 이내에 완료되어야 한다 |

## BIZ 컴포넌트 관리

**화면 목적**: 업무 애플리케이션(BizApp) 컴포넌트를 등록하고 관리하는 화면이다.

**접근 경로**: GNB > 개발 관리 > BIZ 컴포넌트 관리

**화면 유형**: 표준 CRUD 화면 (템플릿 A 참조)

### 화면 레이아웃 구성

| 영역 | 설명 |
|------|------|
| 검색 필터 영역 | 상단, 검색 유형 드롭다운 + 검색어 + 조회 건수 + 조회 버튼 |
| 도구 모음 | 목록 상단 우측, 엑셀 / 출력 버튼 |
| 목록 영역 | 중앙, BIZ 컴포넌트 목록 테이블 |
| 페이지 네비게이션 | 목록 하단 |
| 버튼 영역 | 목록 하단, 등록 버튼 |

### 검색 / 필터

| 필터 항목명 | 입력 유형 | 필수 여부 | 기본값 | 유효성 조건 |
|-------------|-----------|-----------|--------|-------------|
| 검색 유형 | 드롭다운 (컴포넌트명 / 컴포넌트 ID) | 선택 | 컴포넌트명 | - |
| 검색어 | 텍스트 입력 | 선택 | 빈 값 | - |

### 목록 표시

| 컬럼명 | 데이터 유형 | 정렬 가능 | 비고 |
|--------|-------------|-----------|------|
| 컴포넌트 ID | 텍스트 | 가능 | - |
| 컴포넌트명 | 텍스트 | 가능 | - |
| 구현 클래스명 | 텍스트 | 가능 | - |
| 컴포넌트 유형 | 텍스트 | 가능 | - |
| 설명 | 텍스트 | 불가 | - |
| 사용 여부 | 상태 아이콘 | 불가 | Y/N, 초록/빨강 원형 아이콘 |

- 목록 클릭 시 동작: BIZ 컴포넌트 수정 팝업 열림
- 기본 정렬 기준: 컴포넌트 ID 오름차순
- 페이지당 기본 표시 건수: 20건

### 등록 / 수정 팝업

| 항목명 | 입력 유형 | 필수 여부 | 수정 가능 여부 | 유효성 조건 |
|--------|-----------|-----------|----------------|-------------|
| 컴포넌트 ID | 텍스트 입력 | 필수 | 등록 시만 가능 | 영문/숫자, 최대 50자 |
| 컴포넌트명 | 텍스트 입력 | 필수 | 가능 | 최대 100자 |
| 구현 클래스명 | 텍스트 입력 | 필수 | 가능 | 최대 200자 |
| 컴포넌트 유형 | 드롭다운 | 필수 | 가능 | 컴포넌트 유형 목록에서 선택 |
| 설명 | 텍스트 영역 | 선택 | 가능 | 최대 200자 |
| 사용 여부 | 드롭다운 (Y/N) | 필수 | 가능 | 기본값: Y |

### 버튼 및 액션

| 버튼명 | 노출 조건 | 클릭 동작 | 권한 제어 |
|--------|-----------|-----------|-----------|
| 등록 | 항상 노출 | BIZ 컴포넌트 등록 팝업 열기 | 쓰기 권한 |
| 엑셀 | 항상 노출 | 검색 결과 엑셀 다운로드 | 전체 |
| 출력 | 항상 노출 | 목록 인쇄 | 전체 |
| 저장 (팝업) | 팝업 열림 시 | BIZ 컴포넌트 정보 저장 | 쓰기 권한 |
| 삭제 (팝업) | 수정 모드 시 | 확인 팝업 후 BIZ 컴포넌트 삭제 | 쓰기 권한 |

### 개별 기능 요구사항

- 컴포넌트 ID는 등록 시에만 입력 가능하며, 수정 시에는 읽기 전용으로 표시되어야 한다.
- 구현 클래스명은 패키지를 포함한 전체 클래스 경로를 입력할 수 있어야 한다 (예: com.example.biz.SampleComponent).
- 서비스에서 참조 중인 BIZ 컴포넌트를 삭제하려 할 경우 "서비스에서 사용 중인 컴포넌트입니다. 삭제할 수 없습니다." 메시지가 표시되어야 한다.

### 비기능 요구사항

| 항목 | 요구사항 |
|------|----------|
| 응답 속도 | 목록 조회 결과가 3초 이내에 표시되어야 한다 |

## 컴포넌트 관리

**화면 목적**: 서비스에서 사용하는 컴포넌트를 등록하고 파라미터를 설정하는 화면이다.

**접근 경로**: GNB > 개발 관리 > 컴포넌트 관리

**화면 유형**: 표준 CRUD 화면 (템플릿 A 참조)

### 화면 레이아웃 구성

| 영역 | 설명 |
|------|------|
| 검색 필터 영역 | 상단, 검색 유형 드롭다운 + 검색어 + 컴포넌트 유형 드롭다운 + 조회 건수 + 조회 버튼 |
| 도구 모음 | 목록 상단 우측, 엑셀 / 출력 버튼 |
| 목록 영역 | 중앙, 컴포넌트 목록 테이블 |
| 페이지 네비게이션 | 목록 하단 |
| 버튼 영역 | 목록 하단, 등록 버튼 |

### 검색 / 필터

## 전문로그 파싱

**화면 목적**: 전문 데이터를 직접 입력하면 해당 전문의 구조를 파싱하여 분석 결과를 확인하는 화면이다.

**접근 경로**: GNB > 개발 관리 > 전문로그 파싱

**화면 유형**: 특수 화면 (템플릿 D - 입력 후 실행형)

### 화면 레이아웃 구성

| 영역 | 설명 |
|------|------|
| 전문 선택 영역 | 기관 드롭다운 + 전문명 입력(읽기 전용) + 전문 검색 버튼 |
| 전문 데이터 입력 영역 | 전문 원문 데이터를 붙여넣는 대용량 텍스트 영역 |
| 버튼 영역 | 전문파싱 실행 버튼 |

### 입력 항목

| 항목명 | 입력 유형 | 필수 여부 | 유효성 조건 |
|--------|-----------|-----------|-------------|
| 기관 | 드롭다운 | 필수 | 기관 목록에서 선택 |
| 전문명 | 텍스트 (읽기 전용) | 필수 | 전문 검색 팝업에서 선택 후 자동 입력 |
| 전문 데이터 | 텍스트 영역 | 필수 | 파싱할 전문 원문 데이터 |

### 버튼 및 액션

| 버튼명 | 노출 조건 | 클릭 동작 | 권한 제어 |
|--------|-----------|-----------|-----------|
| 전문 검색 | 항상 노출 | 전문 선택 팝업 열기 | 전체 |
| 전문파싱 | 항상 노출 | 입력된 데이터를 파싱하고 결과 팝업 표시 | 전체 |

### 개별 기능 요구사항

- 전문 검색 버튼 클릭 시 팝업에서 전문을 검색하여 선택하면 전문명이 자동으로 입력되어야 한다.
- 전문파싱 실행 시 파싱 결과가 별도 팝업으로 표시되어야 한다.
- 파싱 결과는 전문 구조와 각 필드의 값을 구분하여 표시하여야 한다.

### 비기능 요구사항

| 항목 | 요구사항 |
|------|----------|
| 응답 속도 | 파싱 결과가 3초 이내에 표시되어야 한다 |

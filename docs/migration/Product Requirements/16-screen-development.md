# [개발 관리] 화면 요구사항

---

## 전문로그파싱

**화면 목적**: 전문 데이터를 직접 입력하면 해당 전문의 구조를 파싱하여 분석 결과를 확인하는 화면이다.

**접근 경로**: GNB > 개발 관리 > 전문로그파싱

**화면 유형**: 특수 화면 (템플릿 D - 입력 후 실행형)

### 화면 레이아웃 구성

| 영역 | 설명 |
|------|------|
| 전문 선택 영역 | 기관 드롭다운 + 전문명 입력(읽기 전용) + 전문 검색 버튼 |
| 전문 데이터 입력 영역 | 전문 원문 데이터를 붙여넣는 대용량 텍스트 영역 |
| 버튼 영역 | 전문파싱 실행 버튼 |

### 입력 항목

| 항목명 | 입력 유형 | 필수 여부 | 유효성 조건 |
|--------|-----------|-----------|-------------|
| 기관 | 드롭다운 | 필수 | 기관 목록에서 선택 |
| 전문명 | 텍스트 (읽기 전용) | 필수 | 전문 검색 팝업에서 선택 후 자동 입력 |
| 전문 데이터 | 텍스트 영역 | 필수 | 파싱할 전문 원문 데이터 |

### 버튼 및 액션

| 버튼명 | 노출 조건 | 클릭 동작 | 권한 제어 |
|--------|-----------|-----------|-----------|
| 전문 검색 | 항상 노출 | 전문 선택 팝업 열기 | 전체 |
| 전문파싱 | 항상 노출 | 입력된 데이터를 파싱하고 결과 팝업 표시 | 전체 |

### 개별 기능 요구사항

- 전문 검색 버튼 클릭 시 팝업에서 전문을 검색하여 선택하면 전문명이 자동으로 입력되어야 한다.
- 전문파싱 실행 시 파싱 결과가 별도 팝업으로 표시되어야 한다.
- 파싱 결과는 전문 구조와 각 필드의 값을 구분하여 표시하여야 한다.

### 비기능 요구사항

| 항목 | 요구사항 |
|------|----------|
| 응답 속도 | 파싱 결과가 3초 이내에 표시되어야 한다 |

## 전문로그파싱(JSON)

**화면 목적**: JSON 형식으로 기록된 전문 로그를 조회하고, 파싱된 구조를 확인하는 화면이다.

**접근 경로**: GNB > 개발 관리 > 전문로그파싱(JSON)

**화면 유형**: 조회 전용 화면 (템플릿 B 참조)

### 화면 레이아웃 구성

| 영역 | 설명 |
|------|------|
| 검색 필터 영역 | 검색 조건 입력 영역 |
| 목록 영역 | JSON 전문 로그 목록 |
| 상세 조회 영역 | 선택한 로그의 JSON 파싱 결과 표시 |

### 버튼 및 액션

| 버튼명 | 노출 조건 | 클릭 동작 | 권한 제어 |
|--------|-----------|-----------|-----------|
| 조회 | 항상 노출 | 검색 조건으로 JSON 로그 조회 | 전체 |

### 비기능 요구사항

| 항목 | 요구사항 |
|------|----------|
| 응답 속도 | 조회 결과가 5초 이내에 표시되어야 한다 |

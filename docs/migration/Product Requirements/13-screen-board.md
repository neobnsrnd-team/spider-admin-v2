# [게시판] 화면 요구사항

---

## 게시판 관리

**화면 목적**: 게시판을 생성하고 유형/관리자/카테고리를 설정하는 화면이다.

**접근 경로**: GNB > 게시판 > 게시판 관리

**화면 유형**: 표준 CRUD 화면 (템플릿 A 참조)

### 화면 레이아웃 구성

| 영역 | 설명 |
|------|------|
| 검색 필터 영역 | 상단, 검색 유형 드롭다운 + 검색어 + 조회 건수 + 조회 버튼 |
| 도구 모음 | 목록 상단 우측, 엑셀 / 출력 버튼 |
| 목록 영역 | 중앙, 게시판 목록 테이블 |
| 페이지 네비게이션 | 목록 하단 |
| 버튼 영역 | 목록 하단, 등록 버튼 |

### 검색 / 필터

| 필터 항목명 | 입력 유형 | 필수 여부 | 기본값 | 유효성 조건 |
|-------------|-----------|-----------|--------|-------------|
| 검색 유형 | 드롭다운 (게시판명 / 게시판 ID) | 선택 | 게시판명 | - |
| 검색어 | 텍스트 입력 | 선택 | 빈 값 | - |

### 목록 표시

| 컬럼명 | 데이터 유형 | 정렬 가능 | 비고 |
|--------|-------------|-----------|------|
| 게시판 ID | 텍스트 | 가능 | - |
| 게시판명 | 텍스트 | 가능 | - |
| 유형 | 텍스트 | 가능 | 공지 / 자유 / FAQ |
| 관리자 | 텍스트 | 가능 | 관리자 사용자명 |

- 목록 클릭 시 동작: 게시판 수정 팝업 열림
- 기본 정렬 기준: 게시판 ID 오름차순
- 페이지당 기본 표시 건수: 20건

### 등록 / 수정 팝업

| 항목명 | 입력 유형 | 필수 여부 | 수정 가능 여부 | 유효성 조건 |
|--------|-----------|-----------|----------------|-------------|
| 게시판 ID | 텍스트 입력 | 필수 | 등록 시만 가능 | 영문/숫자, 중복 불가 |
| 게시판명 | 텍스트 입력 | 필수 | 가능 | 최대 50자 |
| 유형 | 드롭다운 (공지 / 자유 / FAQ) | 필수 | 가능 | - |
| 관리자 ID | 드롭다운 | 선택 | 가능 | 사용자 목록에서 선택, API에서 동적 로드 |

### 카테고리 관리 (수정 팝업 내 하단 섹션)

수정 모드에서 해당 게시판의 카테고리 목록이 팝업 하단에 표시되며, 인라인으로 추가/수정/삭제할 수 있다.

| 항목명 | 입력 유형 | 필수 여부 | 유효성 조건 |
|--------|-----------|-----------|-------------|
| 카테고리명 | 텍스트 입력 | 필수 | 최대 50자, 동일 게시판 내 중복 불가 |

### 버튼 및 액션

| 버튼명 | 노출 조건 | 클릭 동작 | 권한 제어 |
|--------|-----------|-----------|-----------|
| 등록 | 항상 노출 | 게시판 등록 팝업 열기 | 쓰기 권한 |
| 엑셀 | 항상 노출 | 검색 결과 엑셀 다운로드 | 전체 |
| 출력 | 항상 노출 | 목록 인쇄 | 전체 |
| 저장 (팝업) | 팝업 열림 시 | 게시판 정보 저장 | 쓰기 권한 |
| 삭제 (팝업) | 수정 모드 시 | 확인 팝업 후 게시판 삭제 | 쓰기 권한 |
| 카테고리 추가 (팝업) | 수정 모드 시 | 카테고리 행 추가 | 쓰기 권한 |
| 카테고리 삭제 (팝업) | 수정 모드, 카테고리 선택 시 | 선택된 카테고리 삭제 | 쓰기 권한 |

### 개별 기능 요구사항

- 수정 모드에서 카테고리를 추가/수정/삭제할 수 있어야 한다.
- 카테고리명은 필수이며, 동일 게시판 내에서 중복될 수 없어야 한다.
- 카테고리명은 최대 50자까지 입력할 수 있어야 한다.
- 게시판 삭제 시 해당 게시판에 게시글이 존재하면 경고 메시지가 표시되어야 한다: "게시판에 게시글이 존재합니다. 삭제하시겠습니까?"
- 관리자 ID 드롭다운은 사용자 목록 API에서 동적으로 로드되어야 한다.

### 비기능 요구사항

| 항목 | 요구사항 |
|------|----------|
| 응답 속도 | 게시판 목록 조회 결과가 3초 이내에 표시되어야 한다 |

## 게시판 권한 관리

**화면 목적**: 게시판별 사용자 읽기/쓰기 권한을 부여하는 화면이다.

**접근 경로**: GNB > 게시판 > 게시판 권한 관리

**화면 유형**: 표준 CRUD 화면 (템플릿 A 참조)

### 화면 레이아웃 구성

| 영역 | 설명 |
|------|------|
| 검색 필터 영역 | 상단, 사용자 필터 + 게시판 필터 + 조회 건수 + 조회 버튼 |
| 도구 모음 | 목록 상단 우측, 엑셀 / 출력 버튼 |
| 목록 영역 | 중앙, 게시판 권한 목록 테이블 |
| 페이지 네비게이션 | 목록 하단 |
| 버튼 영역 | 목록 하단, 등록 버튼 |

### 검색 / 필터

| 필터 항목명 | 입력 유형 | 필수 여부 | 기본값 | 유효성 조건 |
|-------------|-----------|-----------|--------|-------------|
| 사용자 | 드롭다운 (전체 + 사용자 목록) | 선택 | 전체 | 사용자 목록은 API에서 동적 로드 |
| 게시판 | 드롭다운 (전체 + 게시판 목록) | 선택 | 전체 | 게시판 목록은 API에서 동적 로드 |

### 목록 표시

| 컬럼명 | 데이터 유형 | 정렬 가능 | 비고 |
|--------|-------------|-----------|------|
| 사용자 ID | 텍스트 | 가능 | - |
| 사용자명 | 텍스트 | 가능 | 사용자 테이블에서 조인 |
| 게시판명 | 텍스트 | 가능 | 게시판 테이블에서 조인 |
| 권한 | 텍스트 | 가능 | 쓰기 / 읽기전용 |

- 목록 클릭 시 동작: 권한 수정 팝업 열림
- 기본 정렬 기준: 사용자 ID 오름차순
- 페이지당 기본 표시 건수: 20건

### 등록 / 수정 팝업

| 항목명 | 입력 유형 | 필수 여부 | 수정 가능 여부 | 유효성 조건 |
|--------|-----------|-----------|----------------|-------------|
| 사용자 | 드롭다운 | 필수 | 등록 시만 가능 | 사용자 목록에서 선택, API에서 동적 로드 |
| 게시판 | 드롭다운 | 필수 | 등록 시만 가능 | 게시판 목록에서 선택, API에서 동적 로드 |
| 권한 | 라디오 버튼 (쓰기 / 읽기전용) | 필수 | 가능 | 기본값: 읽기전용 |

### 버튼 및 액션

| 버튼명 | 노출 조건 | 클릭 동작 | 권한 제어 |
|--------|-----------|-----------|-----------|
| 등록 | 항상 노출 | 권한 등록 팝업 열기 | 쓰기 권한 |
| 엑셀 | 항상 노출 | 검색 결과 엑셀 다운로드 | 전체 |
| 출력 | 항상 노출 | 목록 인쇄 | 전체 |
| 저장 (팝업) | 팝업 열림 시 | 권한 정보 저장 | 쓰기 권한 |
| 삭제 (팝업) | 수정 모드 시 | 확인 팝업 후 권한 삭제 | 쓰기 권한 |

### 개별 기능 요구사항

- 동일 사용자-게시판 조합에 대해 중복 등록이 불가하여야 한다. 중복 시 "이미 등록된 권한입니다" 메시지가 표시되어야 한다.
- 권한 삭제 시 확인 팝업이 표시되어야 한다: "해당 사용자의 게시판 권한을 삭제하시겠습니까?"

### 비기능 요구사항

| 항목 | 요구사항 |
|------|----------|
| 응답 속도 | 권한 목록 조회 결과가 3초 이내에 표시되어야 한다 |

## 게시판 열람/작성

**화면 목적**: 게시판에서 게시글을 열람하고 작성/수정/삭제하는 화면이다. 게시판 관리에서 생성된 게시판별로 동적으로 메뉴가 생성되어, 각 게시판에 대해 개별 접근이 가능하다.

**접근 경로**: GNB > 게시판 > {게시판명}

**화면 유형**: 특수 화면 (템플릿 D - 게시판형)

### 화면 레이아웃 구성

| 영역 | 설명 |
|------|------|
| 카테고리 필터 영역 | 상단, 카테고리 태그 버튼 (전체 + 게시판에 등록된 카테고리 목록) |
| 검색 영역 | 카테고리 필터 하단, 검색어 입력 + 조회 버튼 |
| 목록 영역 | 중앙, 게시글 목록 테이블 |
| 페이지 네비게이션 | 목록 하단 |
| 버튼 영역 | 목록 하단, 글쓰기 버튼 (쓰기 권한 보유 시에만 노출) |

### 검색 / 필터

| 필터 항목명 | 입력 유형 | 필수 여부 | 기본값 | 유효성 조건 |
|-------------|-----------|-----------|--------|-------------|
| 카테고리 | 태그 버튼 (전체 + 카테고리 목록) | 선택 | 전체 | 게시판에 설정된 카테고리 목록에서 동적 생성 |
| 검색어 | 텍스트 입력 | 선택 | 빈 값 | 제목 + 내용에서 검색 |

### 목록 표시

| 컬럼명 | 데이터 유형 | 정렬 가능 | 비고 |
|--------|-------------|-----------|------|
| 번호 | 숫자 | 가능 | 자동 채번 |
| 카테고리 | 태그 형태 | 가능 | 카테고리명 태그 배지 |
| 제목 | 텍스트 | 가능 | 클릭 시 게시글 상세 표시 |
| 작성자 | 텍스트 | 가능 | 작성자 사용자명 |
| 작성일 | 날짜 | 가능 | yyyy-MM-dd 형식 |
| 조회수 | 숫자 | 가능 | - |

- 목록 클릭 시 동작: 게시글 상세 보기 화면으로 이동
- 기본 정렬 기준: 번호 내림차순 (최신 글 상단)
- 페이지당 기본 표시 건수: 20건

### 게시글 상세 보기

게시글 목록에서 제목을 클릭하면 게시글의 상세 내용이 표시된다.

| 항목명 | 데이터 유형 | 비고 |
|--------|-------------|------|
| 제목 | 텍스트 (읽기 전용) | - |
| 카테고리 | 태그 배지 (읽기 전용) | - |
| 작성자 | 텍스트 (읽기 전용) | 사용자명 |
| 작성일시 | 날짜/시간 (읽기 전용) | yyyy-MM-dd HH:mm |
| 조회수 | 숫자 (읽기 전용) | 상세 진입 시 자동 증가 |
| 내용 | HTML (읽기 전용) | 본문 내용 |
| 첨부파일 | 파일 링크 목록 | 클릭 시 다운로드 |

### 게시글 작성 / 수정 폼

| 항목명 | 입력 유형 | 필수 여부 | 유효성 조건 |
|--------|-----------|-----------|-------------|
| 카테고리 | 드롭다운 | 선택 | 게시판에 카테고리가 설정된 경우에만 표시 |
| 제목 | 텍스트 입력 | 필수 | 최대 200자 |
| 내용 | 텍스트 영역 / 리치텍스트 에디터 | 필수 | 최대 4000자 |
| 첨부파일 | 파일 업로드 | 선택 | 최대 3개, 1개당 최대 10MB |

### 버튼 및 액션

| 버튼명 | 노출 조건 | 클릭 동작 | 권한 제어 |
|--------|-----------|-----------|-----------|
| 글쓰기 | 목록 화면, 쓰기 권한 보유 시 | 게시글 작성 폼 표시 | 쓰기 권한 |
| 저장 | 작성/수정 폼 | 게시글 저장 | 쓰기 권한 |
| 수정 | 상세 보기, 본인 작성 글인 경우 | 게시글 수정 폼 전환 | 쓰기 권한 + 본인 글 |
| 삭제 | 상세 보기, 본인 작성 글인 경우 | 확인 팝업 후 게시글 삭제 | 쓰기 권한 + 본인 글 |
| 목록 | 상세 보기 / 작성 폼 | 게시글 목록으로 이동 | 전체 |
| 취소 | 작성/수정 폼 | 작성 취소, 이전 화면으로 복귀 | 전체 |

### 개별 기능 요구사항

- 카테고리 태그 버튼을 클릭하면 해당 카테고리의 게시글만 필터링되어야 한다.
- "전체" 태그 버튼 클릭 시 모든 게시글이 표시되어야 한다.
- 첨부파일은 최대 3개까지 업로드할 수 있어야 하며, 1개당 최대 10MB로 제한되어야 한다.
- 파일 크기 초과 시 "첨부파일은 1개당 최대 10MB까지 허용됩니다" 메시지가 표시되어야 한다.
- 본인이 작성한 글만 수정/삭제할 수 있어야 한다.
- 쓰기 권한이 없는 사용자에게는 글쓰기 버튼이 표시되지 않아야 한다.
- 게시판 관리에서 생성된 게시판별로 GNB 메뉴에 동적으로 하위 메뉴가 생성되어야 한다.
- 게시글 상세 보기 진입 시 조회수가 자동으로 1 증가하여야 한다.

### 비기능 요구사항

| 항목 | 요구사항 |
|------|----------|
| 응답 속도 | 게시글 목록 조회 결과가 3초 이내에 표시되어야 한다 |
| 첨부파일 제한 | 1개당 최대 10MB, 최대 3개 |

## 긴급공지 관리

**화면 목적**: 다국어별 긴급공지 내용을 인라인 편집하고 미리보기하는 화면이다. 언어별로 독립적인 긴급공지를 작성할 수 있으며, 편집 중인 내용을 실제 팝업 형태로 미리 확인할 수 있다.

**접근 경로**: GNB > 게시판 > 긴급공지 관리

**화면 유형**: 특수 화면 (템플릿 D - 인라인 편집)

**레거시 근거**: FWKM0230.jsp

### 화면 레이아웃 구성

| 영역 | 설명 |
|------|------|
| 좌측 패널 - 언어 목록 | 지원 언어 목록 (한국어, 영어, 일본어, 중국어 등), API에서 지원 언어를 동적 로드 |
| 우측 패널 - 편집 영역 | 선택된 언어의 긴급공지 편집 폼 (인라인 편집) |
| 버튼 영역 | 하단, 저장(일괄) / 미리보기 / 초기화 버튼 |

### 좌측 패널 - 언어 목록

| 컬럼명 | 데이터 유형 | 비고 |
|--------|-------------|------|
| 언어명 | 텍스트 | 한국어, 영어, 일본어, 중국어 등 |
| 수정됨 배지 | 상태 배지 | 변경 사항이 있는 언어에 "수정됨" 배지 표시 |

- 언어 목록은 API에서 시스템이 지원하는 언어를 동적으로 로드하여야 한다.
- 언어를 클릭하면 우측 편집 영역에 해당 언어의 긴급공지 내용이 표시되어야 한다.
- 변경 사항이 있는 언어는 목록에서 "수정됨" 배지로 시각적으로 구분되어야 한다.

### 우측 패널 - 편집 영역

| 항목명 | 입력 유형 | 필수 여부 | 수정 가능 여부 | 유효성 조건 |
|--------|-----------|-----------|----------------|-------------|
| 공지 제목 | 텍스트 입력 | 필수 | 가능 | 최대 100자 |
| 공지 내용 | 텍스트 영역 / 리치텍스트 에디터 | 필수 | 가능 | 최대 2000자 |
| 표시 여부 | 토글 스위치 (Y/N) | 필수 | 가능 | 기본값: N |
| 표시 시작일시 | 날짜/시간 선택 | 선택 | 가능 | 날짜 + 시간(HH:mm) 형식 |
| 표시 종료일시 | 날짜/시간 선택 | 선택 | 가능 | 날짜 + 시간(HH:mm) 형식, 시작일시 이후여야 함 |
| 배경색 | 색상 선택 (컬러 피커) | 선택 | 가능 | 기본값: 노랑(#FFEB3B 또는 유사 색상) |

### 버튼 및 액션

| 버튼명 | 노출 조건 | 클릭 동작 | 권한 제어 |
|--------|-----------|-----------|-----------|
| 저장 | 항상 노출 | 모든 언어의 변경사항을 일괄 저장 | 쓰기 권한 |
| 미리보기 | 항상 노출 | 현재 편집 중인 언어의 긴급공지를 실제 팝업 형태로 표시 | 전체 |
| 초기화 | 항상 노출 | 저장 전 변경사항을 모두 되돌리기 (마지막 저장 상태로 복원) | 전체 |

### 개별 기능 요구사항

- 인라인 편집: 별도의 등록/수정 팝업 없이 우측 편집 영역에서 항목을 직접 수정할 수 있어야 한다.
- 미리보기 버튼 클릭 시 현재 편집 중인 내용이 실제 긴급공지 팝업과 동일한 형태로 모달 위에 표시되어야 한다. 미리보기에는 제목, 내용, 배경색이 반영되어야 한다.
- 언어별 독립 편집: 각 언어의 긴급공지 내용은 개별적으로 작성되며, 다른 언어의 내용에 영향을 주지 않아야 한다.
- 일괄 저장: 저장 버튼 클릭 시 변경된 모든 언어의 긴급공지 내용이 한 번의 요청으로 일괄 저장되어야 한다.
- 변경 사항이 있는 언어는 좌측 언어 목록에서 "수정됨" 배지가 표시되어야 한다. 저장 후에는 배지가 제거되어야 한다.
- 초기화 버튼 클릭 시 확인 팝업이 표시되어야 한다: "변경사항을 모두 되돌리시겠습니까?" 확인 시 마지막 저장 상태로 모든 편집 내용이 복원되어야 한다.
- 표시 종료일시는 표시 시작일시보다 이후여야 한다. 시작일시보다 이전 값을 입력하면 "종료일시는 시작일시 이후여야 합니다" 메시지가 표시되어야 한다.
- 표시 여부가 N인 경우에도 내용을 편집하고 저장할 수 있어야 한다 (미리 작성 후 추후 활성화 용도).
- 저장되지 않은 변경사항이 있는 상태에서 다른 메뉴로 이동하려 하면 "저장되지 않은 변경사항이 있습니다. 이동하시겠습니까?" 경고가 표시되어야 한다.

### 비기능 요구사항

| 항목 | 요구사항 |
|------|----------|
| 응답 속도 | 언어 목록 및 편집 데이터 조회가 3초 이내에 표시되어야 한다 |
| 일괄 저장 | 모든 언어의 변경사항 저장이 3초 이내에 완료되어야 한다 |

## 페이지별 공지사항 관리

**화면 목적**: 시스템의 각 페이지 URL에 표시할 공지사항을 등록하고 관리하는 화면이다.

**접근 경로**: GNB > 게시판 > 페이지별 공지사항 관리

**화면 유형**: 표준 CRUD 화면 (템플릿 A 참조)

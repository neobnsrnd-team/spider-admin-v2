# [홈] 화면 요구사항

---

## 대시보드

**화면 목적**: 로그인 후 최초 진입하는 메인 화면. 시스템 주요 현황 정보를 한눈에 파악할 수 있는 화면이다.

**접근 경로**: 로그인 성공 후 자동 이동 / 상단 로고 클릭

**화면 유형**: 특수 화면 (템플릿 D - 카드 + 목록 조합)

### 화면 레이아웃 구성

| 영역 | 설명 |
|------|------|
| 사용자 정보 영역 | 상단 좌측, 사용자명 + 역할 + 최종 로그인 일시 |
| 현황 요약 카드 영역 | 상단, 시스템 주요 지표를 카드 형태로 4개 수평 배치 |
| 최근 공지사항 영역 | 중단 좌측, 최근 등록된 공지사항 목록 (최대 5건) |
| 바로가기 영역 | 중단 우측, 자주 사용하는 메뉴 바로가기 버튼 |
| 시스템 상태 영역 | 하단, WAS/게이트웨이 간략 상태 요약 |

### 사용자 정보 영역

| 항목명 | 데이터 유형 | 비고 |
|--------|-------------|------|
| 사용자명 | 텍스트 | "OOO님, 환영합니다" 형식 |
| 역할명 | 텍스트 | 현재 사용자의 역할 |
| 최종 로그인 일시 | 날짜/시간 | 이전 로그인 시각 |

### 현황 요약 카드 영역

| 카드명 | 표시 내용 | 아이콘 | 클릭 동작 |
|--------|-----------|--------|-----------|
| 실행 중 배치 | 현재 실행 중인 배치 건수 | 재생 아이콘 | "실행 중인 배치" 화면으로 이동 |
| 오류 발생 | 오늘 발생한 오류 건수 | 경고 아이콘 | "오류 발생 현황" 화면으로 이동 |
| 중지 거래 | 현재 중지 상태인 거래 건수 | 정지 아이콘 | "거래 중지 관리" 화면으로 이동 |
| 전체 사용자 | 활성 사용자 수 | 사용자 아이콘 | "사용자 관리" 화면으로 이동 |

- 각 카드에는 숫자(건수)가 크게 표시되어야 한다.
- 건수가 0이 아닌 경우 강조 색상으로 표시되어야 한다 (오류: 빨강, 중지 거래: 주황).
- 카드 클릭 시 해당 화면의 탭이 열려야 한다.

### 최근 공지사항 영역

| 컬럼명 | 데이터 유형 | 비고 |
|--------|-------------|------|
| 제목 | 텍스트 | 클릭 시 게시글 상세로 이동 |
| 작성자 | 텍스트 | - |
| 작성일 | 날짜 | YYYY-MM-DD |

- 최근 등록된 공지사항 5건이 표시되어야 한다.
- 공지사항이 없을 경우 "등록된 공지사항이 없습니다" 메시지가 표시되어야 한다.
- "더보기" 링크 클릭 시 공지사항 게시판 화면으로 이동되어야 한다.

### 바로가기 영역

- 자주 사용하는 메뉴 바로가기 버튼이 아이콘 + 텍스트 형태로 배치되어야 한다.
- 바로가기 항목은 사용자 역할에 따라 접근 가능한 메뉴만 표시되어야 한다.
- 기본 바로가기 목록: 사용자 관리, 거래 관리, 배치 APP 관리, 오류 발생 현황, 운영정보 Reload
- 바로가기 클릭 시 해당 화면이 탭으로 열려야 한다.

### 시스템 상태 영역

| 항목명 | 데이터 유형 | 비고 |
|--------|-------------|------|
| WAS 인스턴스 상태 | 요약 텍스트 | "활성 N대 / 전체 M대" 형식 |
| 게이트웨이 상태 | 요약 텍스트 | "정상 N개 / 전체 M개" 형식 |

- 비정상 상태가 있을 경우 빨간색으로 강조 표시되어야 한다.
- 상태 텍스트 클릭 시 해당 모니터링 화면으로 이동되어야 한다.

### 개별 기능 요구사항

- 대시보드 데이터는 화면 진입 시 자동으로 조회되어야 한다.
- 수동 새로고침 버튼이 제공되어야 한다.
- 각 영역은 독립적으로 로딩되어야 하며, 일부 영역의 API 호출 실패가 다른 영역에 영향을 미치지 않아야 한다.
- 사용자의 메뉴 접근 권한에 따라 바로가기 항목과 현황 카드가 선택적으로 표시되어야 한다.

### 비기능 요구사항

| 항목 | 요구사항 |
|------|----------|
| 응답 속도 | 대시보드 전체 로딩이 3초 이내에 완료되어야 한다 |
| 데이터 실시간성 | 현황 카드 데이터는 화면 진입 시점의 최신 데이터를 표시하여야 한다 |

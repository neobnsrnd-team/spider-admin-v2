# [배치 관리] 화면 요구사항

---

## 배치 APP 관리

**화면 목적**: 배치 애플리케이션을 등록/관리하고, WAS 인스턴스를 할당하며, 수동 실행을 요청하는 화면이다.

**접근 경로**: GNB > 배치 관리 > 배치 APP 관리

**화면 유형**: 표준 CRUD 화면 (템플릿 A 참조)

### 화면 레이아웃 구성

| 영역 | 설명 |
|------|------|
| 검색 필터 영역 | 상단, 검색 유형 + 검색어 + 인스턴스 드롭다운 + 조회 건수 + 조회 버튼 |
| 도구 모음 | 목록 상단 우측, 엑셀 / 출력 버튼 |
| 목록 영역 | 중앙, 배치 APP 목록 테이블 |
| 페이지 네비게이션 | 목록 하단 |
| 버튼 영역 | 목록 하단, 등록 버튼 |

### 검색 / 필터

| 필터 항목명 | 입력 유형 | 필수 여부 | 기본값 | 유효성 조건 |
|-------------|-----------|-----------|--------|-------------|
| 검색 유형 | 드롭다운 (배치 APP명 / 배치 APP ID) | 선택 | 배치 APP명 | - |
| 검색어 | 텍스트 입력 | 선택 | 빈 값 | - |
| 인스턴스 ID | 드롭다운 (전체 + 인스턴스 목록) | 선택 | 전체 | API에서 동적 로드 |

### 목록 표시

| 컬럼명 | 데이터 유형 | 정렬 가능 | 비고 |
|--------|-------------|-----------|------|
| 배치 APP ID | 텍스트 | 가능 | - |
| 배치 APP명 | 텍스트 | 가능 | - |
| 파일명 | 텍스트 | 가능 | - |
| 선행 배치 APP ID | 텍스트 | 불가 | - |
| 실행 주기 | 텍스트 | 가능 | 매일~수동 |
| 재실행 | 상태 아이콘 | 불가 | Y/N |
| 다중 WAS | 상태 아이콘 | 불가 | Y/N |
| 중요도 | 상태 배지 | 가능 | 높음=빨강, 보통=노랑, 낮음=회색 |
| 수동 실행 | 버튼 (▶) | 불가 | 클릭 시 수동 실행 팝업 |

- 목록 클릭 시 동작: 배치 APP 수정 팝업 열림
- 기본 정렬 기준: 배치 APP ID 오름차순
- 페이지당 기본 표시 건수: 20건

### 등록 / 수정 팝업 (최대 너비 1000px)

**기본 정보**

| 항목명 | 입력 유형 | 필수 여부 | 수정 가능 여부 | 유효성 조건 |
|--------|-----------|-----------|----------------|-------------|
| 배치 APP ID | 텍스트 입력 | 필수 | 등록 시만 가능 | 최대 50자 |
| APP명 | 텍스트 입력 | 선택 | 가능 | 최대 50자 |
| 파일명 | 텍스트 입력 | 필수 | 가능 | 최대 200자 |
| 설명 | 텍스트 영역 | 선택 | 가능 | 최대 200자 |
| 파라미터 | 텍스트 입력 | 선택 | 가능 | 최대 500자, key=value 형식 |
| 선행 배치 | 드롭다운 | 선택 | 가능 | 배치 APP 목록에서 선택 |
| CRON | 텍스트 입력 | 선택 | 가능 | 최대 20자, 도움말 버튼 제공 |

**WAS 인스턴스 할당**

- 항목 이동 위젯을 사용하여 WAS 인스턴스를 할당/해제
- 우측 패널에 인스턴스 ID, 설명, IP, PORT, 유형 정보 표시

**부가 정보 (접기/펼치기)**

| 항목명 | 입력 유형 | 필수 여부 | 수정 가능 여부 | 유효성 조건 |
|--------|-----------|-----------|----------------|-------------|
| 다중 WAS | 드롭다운 (Y/N) | 선택 | 가능 | - |
| 실행 주기 | 드롭다운 | 선택 | 가능 | - |
| 재실행 | 드롭다운 (Y/N) | 선택 | 가능 | - |
| 중요도 | 드롭다운 | 선택 | 가능 | - |
| 관련 기관 | 드롭다운 | 선택 | 가능 | - |
| 관련 거래 | 드롭다운 | 선택 | 가능 | - |
| 입출력 구분 | 드롭다운 | 선택 | 가능 | - |

### 개별 기능 요구사항

- 수동 실행 버튼(▶) 클릭 시 기준일, 파라미터, 인스턴스를 선택하는 팝업이 표시되어야 한다.
- CRON 필드 옆에 도움말 버튼이 제공되어야 한다.
- 부가 정보 영역은 접기/펼치기가 가능하여야 한다.

### 비기능 요구사항

| 항목 | 요구사항 |
|------|----------|
| 응답 속도 | 목록 조회 결과가 3초 이내에 표시되어야 한다 |
| 수동 실행 | 수동 실행 요청 후 결과가 10초 이내에 표시되어야 한다 |

## 배치 수행 내역

**화면 목적**: 배치 실행 이력과 결과를 조회하는 화면이다.

**접근 경로**: GNB > 배치 관리 > 배치 수행 내역

**화면 유형**: 조회 전용 화면 (템플릿 B 참조)

### 화면 레이아웃 구성

| 영역 | 설명 |
|------|------|
| 검색 필터 영역 | 상단, 배치 APP ID + 인스턴스 + 상태 + 기준일 + 조회 건수 + 조회 버튼 |
| 도구 모음 | 목록 상단 우측, 엑셀 / 출력 버튼 |
| 목록 영역 | 중앙, 배치 수행 내역 테이블 |
| 페이지 네비게이션 | 목록 하단 |

### 검색 / 필터

| 필터 항목명 | 입력 유형 | 필수 여부 | 기본값 | 유효성 조건 |
|-------------|-----------|-----------|--------|-------------|
| 배치 APP ID | 텍스트 입력 | 선택 | 빈 값 | - |
| 인스턴스 | 드롭다운 (전체 + 인스턴스 목록) | 선택 | 전체 | - |
| 상태 | 드롭다운 (전체/요청됨/시작됨/성공/실패) | 선택 | 전체 | - |
| 기준일 | 날짜 선택 | 선택 | 오늘 | - |

### 목록 표시

| 컬럼명 | 데이터 유형 | 정렬 가능 | 비고 |
|--------|-------------|-----------|------|
| 배치 APP ID | 텍스트 | 가능 | - |
| 인스턴스 ID | 텍스트 | 가능 | - |
| 실행 횟수 | 숫자 | 가능 | - |
| 기준일 | 날짜 | 가능 | - |
| 시작 일시 | 날짜/시간 | 가능 | - |
| 종료 일시 | 날짜/시간 | 가능 | - |
| 상태 | 상태 배지 | 불가 | 성공=초록, 시작됨=파랑, 실패=빨강, 요청됨=노랑 |
| 수행자 ID | 텍스트 | 불가 | - |
| 총 처리건 | 숫자 | 불가 | - |
| 정상 처리건 | 숫자 | 불가 | - |
| 오류 처리건 | 숫자 | 불가 | - |
| 오류 코드 | 텍스트 | 불가 | - |

- 기본 정렬 기준: 시작 일시 내림차순
- 페이지당 기본 표시 건수: 20건

### 버튼 및 액션

| 버튼명 | 노출 조건 | 클릭 동작 | 권한 제어 |
|--------|-----------|-----------|-----------|
| 엑셀 | 항상 노출 | 검색 결과 엑셀 다운로드 | 전체 |
| 출력 | 항상 노출 | 목록 인쇄 | 전체 |

### 비기능 요구사항

| 항목 | 요구사항 |
|------|----------|
| 응답 속도 | 목록 조회 결과가 3초 이내에 표시되어야 한다 |

## 실행 중인 배치

**화면 목적**: 현재 실행 중이거나 대기 중인 배치를 모니터링하는 화면이다.

**접근 경로**: GNB > 배치 관리 > 실행 중인 배치

**화면 유형**: 모니터링 화면 (템플릿 C 참조)

### 화면 레이아웃 구성

| 영역 | 설명 |
|------|------|
| 제어 영역 | 상단, 자동 새로고침 ON/OFF + 수동 새로고침 버튼 |
| 목록 영역 | 중앙, 실행 중인 배치 목록 테이블 |

### 목록 표시

| 컬럼명 | 데이터 유형 | 정렬 가능 | 비고 |
|--------|-------------|-----------|------|
| 배치 APP ID | 텍스트 | 불가 | - |
| 파일명 | 텍스트 | 불가 | - |
| CRON | 텍스트 | 불가 | - |
| 인스턴스 ID | 텍스트 | 불가 | - |
| 시작 일시 | 날짜/시간 | 불가 | - |
| 상태 | 상태 배지 | 불가 | 시작됨=파랑, 요청됨=노랑 |

### 버튼 및 액션

| 버튼명 | 노출 조건 | 클릭 동작 | 권한 제어 |
|--------|-----------|-----------|-----------|
| 수동 새로고침 | 항상 노출 | 목록 즉시 갱신 | 전체 |
| 자동 새로고침 ON/OFF | 항상 노출 | 자동 갱신 활성화/비활성화 토글 | 전체 |

### 개별 기능 요구사항

- 자동 새로고침 기능이 제공되어야 한다.

### 비기능 요구사항

| 항목 | 요구사항 |
|------|----------|
| 갱신 주기 | 10초 간격으로 자동 갱신되어야 한다 |
| 응답 속도 | 목록 갱신이 2초 이내에 완료되어야 한다 |

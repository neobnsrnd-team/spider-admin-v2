# [시스템 관리] 화면 요구사항

---

## 메뉴 관리

**화면 목적**: 시스템의 전체 메뉴 구조를 조회하고, 메뉴를 등록/수정/삭제하며, 메뉴 간 계층(부모-자식) 관계를 설정하는 화면이다.

**접근 경로**: GNB > 시스템 관리 > 메뉴 관리

**화면 유형**: 표준 CRUD 화면 (템플릿 A 참조)

### 화면 레이아웃 구성

| 영역 | 설명 |
|------|------|
| 검색 필터 영역 | 상단, 검색 유형 드롭다운 + 검색어 + 메뉴 URL + 상위 메뉴 ID + 조회 건수 + 조회 버튼 |
| 도구 모음 | 목록 상단 우측, 엑셀 / 출력 버튼 |
| 목록 영역 | 중앙, 메뉴 목록 테이블 |
| 페이지 네비게이션 | 목록 하단 |
| 버튼 영역 | 목록 하단, 등록 버튼 |

### 검색 / 필터

| 필터 항목명 | 입력 유형 | 필수 여부 | 기본값 | 유효성 조건 |
|-------------|-----------|-----------|--------|-------------|
| 검색 유형 | 드롭다운 (메뉴명 / 메뉴 ID) | 선택 | 메뉴명 | - |
| 검색어 | 텍스트 입력 | 선택 | 빈 값 | - |
| 메뉴 URL | 텍스트 입력 | 선택 | 빈 값 | - |
| 상위 메뉴 ID | 텍스트 입력 + 검색 버튼 | 선택 | 빈 값 | 검색 버튼 클릭 시 상위 메뉴 트리 선택 팝업 표시 |

### 목록 표시

| 컬럼명 | 데이터 유형 | 정렬 가능 | 비고 |
|--------|-------------|-----------|------|
| 메뉴 ID | 텍스트 | 가능 | 너비 180px |
| 메뉴명 | 텍스트 | 가능 | 너비 200px |
| 메뉴 URL | 텍스트 | 가능 | 너비 350px, 좌측 정렬 |
| 표시 여부 | 상태 아이콘 | 불가 | 초록/빨강 원형 아이콘 |
| 사용 여부 | 상태 아이콘 | 불가 | 초록/빨강 원형 아이콘 |
| 하위 메뉴 | 버튼 | 불가 | "보기" 버튼 클릭 시 하위 메뉴 목록 팝업 |

- 목록 클릭 시 동작: 해당 메뉴의 수정 팝업 열림
- 다중 선택 지원 여부: 미지원
- 기본 정렬 기준: 메뉴 ID 오름차순
- 페이지당 기본 표시 건수: 20건

### 등록 / 수정 팝업

| 항목명 | 입력 유형 | 필수 여부 | 수정 가능 여부 | 유효성 조건 |
|--------|-----------|-----------|----------------|-------------|
| 메뉴 ID | 텍스트 입력 | 필수 | 등록 시만 가능 | 영문/숫자, 최대 50자 |
| 메뉴명 | 텍스트 입력 | 필수 | 가능 | 최대 50자 |
| 상위 메뉴 ID | 텍스트 입력 + 검색 | 선택 | 가능 | 트리 선택 팝업으로 입력 |
| 정렬 순서 | 숫자 입력 | 선택 | 가능 | 0~999 |
| 메뉴 URL | 텍스트 영역 | 필수 | 가능 | 최대 200자 |
| 메뉴 아이콘 | 텍스트 입력 | 선택 | 가능 | 최대 50자 |
| 웹앱 ID | 텍스트 입력 | 선택 | 가능 | 최대 50자 |
| 표시 여부 | 드롭다운 (Y/N) | 필수 | 가능 | 기본값: Y |
| 사용 여부 | 드롭다운 (Y/N) | 필수 | 가능 | 기본값: Y |

### 버튼 및 액션

| 버튼명 | 노출 조건 | 클릭 동작 | 권한 제어 |
|--------|-----------|-----------|-----------|
| 등록 | 항상 노출 | 메뉴 등록 팝업 열기 | 쓰기 권한 보유자만 |
| 엑셀 | 항상 노출 | 검색 결과 엑셀 다운로드 | 전체 |
| 출력 | 항상 노출 | 목록 인쇄 | 전체 |
| 저장 (팝업) | 팝업 열림 시 | 메뉴 정보 저장 | 쓰기 권한 |
| 삭제 (팝업) | 수정 모드 시 | 확인 팝업 후 메뉴 삭제 | 쓰기 권한 |
| 하위 메뉴 보기 | 목록 행 내 | 해당 메뉴의 하위 메뉴 목록 팝업 | 전체 |

### 개별 기능 요구사항

- 상위 메뉴 ID 검색 버튼 클릭 시 트리 형태의 메뉴 선택 팝업이 표시되어야 한다.
- 트리 선택 팝업에서 "루트 메뉴로 등록" 옵션이 제공되어야 한다.
- 하위 메뉴 보기 팝업에서 이전/다음 네비게이션으로 하위 메뉴 간 이동이 가능하여야 한다.
- 하위 메뉴 보기 팝업에서 "X / Y 건" 형식으로 현재 위치가 표시되어야 한다.

### 비기능 요구사항

| 항목 | 요구사항 |
|------|----------|
| 응답 속도 | 메뉴 목록 조회 결과가 3초 이내에 표시되어야 한다 |

## 역할 관리

**화면 목적**: 사용자에게 부여할 역할(그룹)을 정의하고, 각 역할에 접근 가능한 메뉴 권한을 설정하는 화면이다.

**접근 경로**: GNB > 시스템 관리 > 역할 관리

**화면 유형**: 표준 CRUD 화면 (템플릿 A 참조)

### 화면 레이아웃 구성

| 영역 | 설명 |
|------|------|
| 검색 필터 영역 | 상단, 역할명/역할 ID 검색 |
| 도구 모음 | 엑셀 / 출력 버튼 |
| 목록 영역 | 역할 목록 테이블 |
| 페이지 네비게이션 | 목록 하단 |
| 버튼 영역 | 등록 버튼 |

### 검색 / 필터

| 필터 항목명 | 입력 유형 | 필수 여부 | 기본값 | 유효성 조건 |
|-------------|-----------|-----------|--------|-------------|
| 검색 유형 | 드롭다운 (역할명 / 역할 ID) | 선택 | 역할명 | - |
| 검색어 | 텍스트 입력 | 선택 | 빈 값 | - |

### 목록 표시

| 컬럼명 | 데이터 유형 | 정렬 가능 | 비고 |
|--------|-------------|-----------|------|
| 역할 ID | 텍스트 | 가능 | - |
| 역할명 | 텍스트 | 가능 | - |
| 역할 설명 | 텍스트 | 가능 | - |
| 사용 여부 | 상태 아이콘 | 가능 | Y/N 배지 |
| 사용자 수 | 숫자 | 가능 | 해당 역할에 할당된 사용자 수 |
| 최종 수정일시 | 날짜/시간 | 가능 | - |

- 목록 클릭 시 동작: 역할 수정 팝업 열림
- 기본 정렬 기준: 역할 ID 오름차순
- 페이지당 기본 표시 건수: 20건

### 등록 / 수정 팝업

| 항목명 | 입력 유형 | 필수 여부 | 수정 가능 여부 | 유효성 조건 |
|--------|-----------|-----------|----------------|-------------|
| 역할 ID | 텍스트 입력 | 필수 | 등록 시만 가능 | 영문/숫자 |
| 역할명 | 텍스트 입력 | 필수 | 가능 | 최대 50자, 중복 불가 |
| 역할 설명 | 텍스트 영역 | 선택 | 가능 | 최대 200자 |
| 사용 여부 | 드롭다운 (Y/N) | 필수 | 가능 | 기본값: Y |

### 버튼 및 액션

| 버튼명 | 노출 조건 | 클릭 동작 | 권한 제어 |
|--------|-----------|-----------|-----------|
| 등록 | 항상 노출 | 역할 등록 팝업 열기 | 쓰기 권한 |
| 메뉴 권한 설정 | 수정 팝업 내 | 역할-메뉴 권한 설정 팝업 열기 | 쓰기 권한 |
| 저장 (팝업) | 팝업 열림 시 | 역할 정보 저장 | 쓰기 권한 |
| 삭제 (팝업) | 수정 모드 시 | 확인 팝업 후 역할 삭제 | 쓰기 권한 |

### 개별 기능 요구사항

- 메뉴 권한 설정 팝업에서 항목 이동 위젯을 사용하여 메뉴를 할당/해제할 수 있어야 한다.
- 각 메뉴에 대해 읽기/쓰기 권한 수준을 개별 설정할 수 있어야 한다.
- 역할명 중복 시 등록/수정이 거부되어야 하며, "이미 존재하는 역할명입니다" 메시지가 표시되어야 한다.
- 사용자가 할당된 역할은 삭제 시 경고 메시지가 표시되어야 한다.

### 비기능 요구사항

| 항목 | 요구사항 |
|------|----------|
| 응답 속도 | 역할 목록 조회 결과가 3초 이내에 표시되어야 한다 |

## 사용자 관리

**화면 목적**: 내부 시스템 사용자 계정을 조회, 등록, 수정, 삭제하고 역할을 부여하는 화면이다.

**접근 경로**: GNB > 시스템 관리 > 사용자 관리

**화면 유형**: 표준 CRUD 화면 (템플릿 A 참조)

### 화면 레이아웃 구성

| 영역 | 설명 |
|------|------|
| 검색 필터 영역 | 상단, 검색 유형 + 검색어 + 역할 필터 + 분류 필터 + 상태 필터 + 조회 건수 |
| 도구 모음 | 엑셀 / 출력 버튼 |
| 목록 영역 | 사용자 목록 테이블 |
| 페이지 네비게이션 | 목록 하단 |
| 버튼 영역 | 등록 버튼 |

### 검색 / 필터

| 필터 항목명 | 입력 유형 | 필수 여부 | 기본값 | 유효성 조건 |
|-------------|-----------|-----------|--------|-------------|
| 검색 유형 | 드롭다운 (사용자명 / 이메일) | 선택 | 사용자명 | - |
| 검색어 | 텍스트 입력 | 선택 | 빈 값 | - |
| 역할 | 드롭다운 (전체 + 역할 목록) | 선택 | 전체 | 역할 목록은 API에서 동적 로드 |
| 분류 | 드롭다운 (전체 + 분류 목록) | 선택 | 전체 | - |
| 사용자 상태 | 드롭다운 (전체/정상/삭제/정지) | 선택 | 전체 | - |

### 목록 표시

| 컬럼명 | 데이터 유형 | 정렬 가능 | 비고 |
|--------|-------------|-----------|------|
| 사용자 ID | 텍스트 | 가능 | - |
| 사용자명 | 텍스트 | 가능 | - |
| 역할명 | 텍스트 | 가능 | 역할 테이블에서 조인 |
| 이메일 | 텍스트 | 가능 | - |
| 분류 | 텍스트 | 가능 | - |
| 상태 | 상태 배지 | 가능 | 정상/삭제/정지 |
| 접근 IP | 텍스트 | 불가 | - |
| 최종 수정일시 | 날짜/시간 | 가능 | - |

- 목록 클릭 시 동작: 사용자 수정 팝업 열림
- 기본 정렬 기준: 사용자 ID 오름차순
- 페이지당 기본 표시 건수: 20건

### 등록 / 수정 팝업

| 항목명 | 입력 유형 | 필수 여부 | 수정 가능 여부 | 유효성 조건 |
|--------|-----------|-----------|----------------|-------------|
| 사용자 ID | 텍스트 입력 | 필수 | 등록 시만 가능 | 영문/숫자, 중복 불가 |
| 사용자명 | 텍스트 입력 | 필수 | 가능 | 최대 50자, 중복 불가 |
| 비밀번호 | 비밀번호 입력 | 등록 시 필수 | 가능 | 최대 50자 |
| 역할 | 드롭다운 | 필수 | 가능 | 활성 역할 목록에서 선택 |
| 이메일 | 텍스트 입력 | 선택 | 가능 | 이메일 형식, 최대 100자, 중복 불가 |
| 전화번호 | 텍스트 입력 | 선택 | 가능 | 최대 13자 |
| 분류 | 드롭다운 | 선택 | 가능 | - |
| 직위 | 텍스트 입력 | 선택 | 가능 | 최대 100자 |
| 주소 | 텍스트 입력 | 선택 | 가능 | 최대 200자 |
| 접근 IP | 텍스트 입력 | 선택 | 가능 | 최대 15자, IP 형식 |
| 사용자 상태 | 드롭다운 (정상/삭제/정지) | 필수 | 가능 | 기본값: 정상 |
| 주민번호 | 텍스트 입력 | 선택 | 가능 | 최대 13자 |
| 사번 | 텍스트 입력 | 선택 | 가능 | 최대 10자 |
| 부서번호 | 텍스트 입력 | 선택 | 가능 | 최대 10자 |
| 업무 권한 코드 | 텍스트 입력 | 선택 | 가능 | 최대 100자 |

### 버튼 및 액션

| 버튼명 | 노출 조건 | 클릭 동작 | 권한 제어 |
|--------|-----------|-----------|-----------|
| 등록 | 항상 노출 | 사용자 등록 팝업 열기 | 쓰기 권한 |
| 저장 (팝업) | 팝업 열림 시 | 사용자 정보 저장 | 쓰기 권한 |
| 삭제 (팝업) | 수정 모드 시 | 확인 팝업 후 사용자 삭제 | 쓰기 권한 |

### 개별 기능 요구사항

- 사용자 ID, 사용자명, 이메일은 등록 시 중복 여부가 실시간으로 확인되어야 한다.
- 중복된 값 입력 시 해당 항목 하단에 "이미 사용 중인 값입니다" 메시지가 표시되어야 한다.
- 수정 시 비밀번호 필드를 비워두면 기존 비밀번호가 유지되어야 한다.

### 비기능 요구사항

| 항목 | 요구사항 |
|------|----------|
| 응답 속도 | 사용자 목록 조회 결과가 3초 이내에 표시되어야 한다 |

## 코드 그룹 관리

**화면 목적**: 시스템에서 사용하는 공통 코드를 분류하는 코드 그룹을 조회, 등록, 수정, 삭제하는 화면이다.

**접근 경로**: GNB > 시스템 관리 > 코드 그룹 관리

**화면 유형**: 표준 CRUD 화면 (템플릿 A 참조)

### 화면 레이아웃 구성

| 영역 | 설명 |
|------|------|
| 검색 필터 영역 | 상단, 검색 유형 드롭다운 + 검색어 + 조회 건수 + 조회 버튼 |
| 도구 모음 | 목록 상단 우측, 엑셀 / 출력 버튼 |
| 목록 영역 | 중앙, 코드 그룹 목록 테이블 |
| 페이지 네비게이션 | 목록 하단 |
| 버튼 영역 | 목록 하단, 등록 버튼 |

### 검색 / 필터

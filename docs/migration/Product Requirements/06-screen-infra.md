# [인프라 관리] 화면 요구사항

---

## WAS 그룹 관리

**화면 목적**: WAS 서버 그룹을 등록하고 관리하는 화면이다. WAS 인스턴스를 논리적으로 그룹화한다.

**접근 경로**: GNB > 인프라 관리 > WAS 그룹 관리

**화면 유형**: 표준 CRUD 화면 (템플릿 A 참조)

### 화면 레이아웃 구성

| 영역 | 설명 |
|------|------|
| 검색 필터 영역 | 상단, 검색 유형 드롭다운 + 검색어 + 조회 건수 + 조회 버튼 |
| 도구 모음 | 목록 상단 우측, 엑셀 / 출력 버튼 |
| 목록 영역 | 중앙, WAS 그룹 목록 테이블 |
| 페이지 네비게이션 | 목록 하단 |
| 버튼 영역 | 목록 하단, 등록 버튼 |

### 검색 / 필터

| 필터 항목명 | 입력 유형 | 필수 여부 | 기본값 | 유효성 조건 |
|-------------|-----------|-----------|--------|-------------|
| 검색 유형 | 드롭다운 (그룹명 / 그룹 ID) | 선택 | 그룹명 | - |
| 검색어 | 텍스트 입력 | 선택 | 빈 값 | - |

### 목록 표시

| 컬럼명 | 데이터 유형 | 정렬 가능 | 비고 |
|--------|-------------|-----------|------|
| WAS 그룹 ID | 텍스트 | 가능 | - |
| WAS 그룹명 | 텍스트 | 가능 | - |
| 설명 | 텍스트 | 가능 | - |
| 인스턴스 수 | 숫자 | 가능 | 그룹에 속한 인스턴스 수 |

- 목록 클릭 시 동작: WAS 그룹 수정 팝업 열림
- 기본 정렬 기준: WAS 그룹 ID 오름차순
- 페이지당 기본 표시 건수: 20건

### 등록 / 수정 팝업

| 항목명 | 입력 유형 | 필수 여부 | 수정 가능 여부 | 유효성 조건 |
|--------|-----------|-----------|----------------|-------------|
| WAS 그룹 ID | 텍스트 입력 | 필수 | 등록 시만 가능 | 영문/숫자 |
| WAS 그룹명 | 텍스트 입력 | 필수 | 가능 | 최대 50자 |
| 설명 | 텍스트 영역 | 선택 | 가능 | 최대 200자 |

### 버튼 및 액션

| 버튼명 | 노출 조건 | 클릭 동작 | 권한 제어 |
|--------|-----------|-----------|-----------|
| 등록 | 항상 노출 | WAS 그룹 등록 팝업 열기 | 쓰기 권한 |
| 저장 (팝업) | 팝업 열림 시 | WAS 그룹 저장 | 쓰기 권한 |
| 삭제 (팝업) | 수정 모드 시 | 확인 팝업 후 삭제 | 쓰기 권한 |

### 비기능 요구사항

| 항목 | 요구사항 |
|------|----------|
| 응답 속도 | 목록 조회 결과가 3초 이내에 표시되어야 한다 |

## WAS 인스턴스

**화면 목적**: 개별 WAS 인스턴스의 정보(IP, 포트, 유형 등)를 등록하고 관리하며, 인스턴스별 속성을 설정하는 화면이다.

**접근 경로**: GNB > 인프라 관리 > WAS 인스턴스

**화면 유형**: 표준 CRUD 화면 (템플릿 A 참조)

### 화면 레이아웃 구성

| 영역 | 설명 |
|------|------|
| 검색 필터 영역 | 상단, 검색 유형 + 검색어 + WAS 그룹 드롭다운 + 조회 건수 + 조회 버튼 |
| 도구 모음 | 목록 상단 우측, 엑셀 / 출력 버튼 |
| 목록 영역 | 중앙, WAS 인스턴스 목록 테이블 |
| 페이지 네비게이션 | 목록 하단 |
| 버튼 영역 | 목록 하단, 등록 버튼 |

### 검색 / 필터

| 필터 항목명 | 입력 유형 | 필수 여부 | 기본값 | 유효성 조건 |
|-------------|-----------|-----------|--------|-------------|
| 검색 유형 | 드롭다운 (인스턴스 ID / 인스턴스 IP) | 선택 | 인스턴스 ID | - |
| 검색어 | 텍스트 입력 | 선택 | 빈 값 | - |
| WAS 그룹 | 드롭다운 (전체 + 그룹 목록) | 선택 | 전체 | API에서 동적 로드 |

### 목록 표시

| 컬럼명 | 데이터 유형 | 정렬 가능 | 비고 |
|--------|-------------|-----------|------|
| 인스턴스 ID | 텍스트 | 가능 | - |
| 인스턴스 설명 | 텍스트 | 가능 | - |
| IP | 텍스트 | 가능 | - |
| 포트 | 숫자 | 가능 | - |
| 유형 | 텍스트 | 가능 | - |
| WAS 그룹 | 텍스트 | 가능 | - |
| 사용 여부 | 상태 아이콘 | 가능 | Y/N |

- 목록 클릭 시 동작: WAS 인스턴스 수정 팝업 열림
- 기본 정렬 기준: 인스턴스 ID 오름차순
- 페이지당 기본 표시 건수: 20건

### 등록 / 수정 팝업

| 항목명 | 입력 유형 | 필수 여부 | 수정 가능 여부 | 유효성 조건 |
|--------|-----------|-----------|----------------|-------------|
| 인스턴스 ID | 텍스트 입력 | 필수 | 등록 시만 가능 | 영문/숫자 |
| 인스턴스 설명 | 텍스트 입력 | 선택 | 가능 | 최대 200자 |
| IP | 텍스트 입력 | 필수 | 가능 | IP 형식 (x.x.x.x) |
| 포트 | 숫자 입력 | 필수 | 가능 | 1~65535 |
| 유형 | 드롭다운 | 필수 | 가능 | - |
| WAS 그룹 | 드롭다운 | 필수 | 가능 | WAS 그룹 목록에서 선택 |
| 사용 여부 | 드롭다운 (Y/N) | 필수 | 가능 | 기본값: Y |

### 개별 기능 요구사항

- 수정 팝업에서 해당 인스턴스의 속성(Property) 목록이 하단 섹션에 표시되어야 한다.
- 인스턴스별 속성을 추가/수정/삭제할 수 있어야 한다.

### 비기능 요구사항

| 항목 | 요구사항 |
|------|----------|
| 응답 속도 | 목록 조회 결과가 3초 이내에 표시되어야 한다 |

## 프로퍼티 DB 관리

**화면 목적**: 시스템 운영에 필요한 속성(Property) 값을 관리하고, WAS 인스턴스별 반영 이력을 추적하는 화면이다.

**접근 경로**: GNB > 인프라 관리 > 프로퍼티 DB 관리

**화면 유형**: 표준 CRUD 화면 (템플릿 A 참조)

### 화면 레이아웃 구성

| 영역 | 설명 |
|------|------|
| 검색 필터 영역 | 상단, 검색 유형 + 검색어 + 속성 그룹 드롭다운 + 조회 건수 + 조회 버튼 |
| 도구 모음 | 목록 상단 우측, 엑셀 / 출력 버튼 |
| 목록 영역 | 중앙, 속성 목록 테이블 |
| 페이지 네비게이션 | 목록 하단 |
| 버튼 영역 | 목록 하단, 등록 버튼 |

### 검색 / 필터

| 필터 항목명 | 입력 유형 | 필수 여부 | 기본값 | 유효성 조건 |
|-------------|-----------|-----------|--------|-------------|
| 검색 유형 | 드롭다운 (속성명 / 속성 ID) | 선택 | 속성명 | - |
| 검색어 | 텍스트 입력 | 선택 | 빈 값 | - |
| 속성 그룹 | 드롭다운 | 선택 | 전체 | - |

### 목록 표시

| 컬럼명 | 데이터 유형 | 정렬 가능 | 비고 |

| 속성 ID | 텍스트 | 가능 | - |
| 속성명 | 텍스트 | 가능 | - |
| 속성값 | 텍스트 | 가능 | - |
| 그룹 | 텍스트 | 가능 | - |
| 설명 | 텍스트 | 가능 | - |

- 목록 클릭 시 동작: 속성 상세 팝업 열림
- 기본 정렬 기준: 속성 ID 오름차순
- 페이지당 기본 표시 건수: 20건

### 등록 / 수정 팝업

| 항목명 | 입력 유형 | 필수 여부 | 수정 가능 여부 | 유효성 조건 |
|--------|-----------|-----------|----------------|-------------|
| 속성 ID | 텍스트 입력 | 필수 | 등록 시만 가능 | 영문/숫자/점(.), 최대 100자 |
| 속성명 | 텍스트 입력 | 필수 | 가능 | 최대 100자 |
| 속성값 | 텍스트 영역 | 필수 | 가능 | 최대 500자 |
| 속성 그룹 | 드롭다운 | 선택 | 가능 | 속성 그룹 목록에서 선택 |
| 설명 | 텍스트 영역 | 선택 | 가능 | 최대 200자 |
| 사용 여부 | 드롭다운 (Y/N) | 필수 | 가능 | 기본값: Y |

### 버튼 및 액션

| 버튼명 | 노출 조건 | 클릭 동작 | 권한 제어 |
|--------|-----------|-----------|-----------|
| 등록 | 항상 노출 | 속성 등록 팝업 열기 | 쓰기 권한 |
| 엑셀 | 항상 노출 | 검색 결과 엑셀 다운로드 | 전체 |
| 출력 | 항상 노출 | 목록 인쇄 | 전체 |
| 저장 (팝업) | 팝업 열림 시 | 속성 정보 저장 | 쓰기 권한 |
| 삭제 (팝업) | 수정 모드 시 | 확인 팝업 후 삭제 | 쓰기 권한 |
| Reload | 수정 팝업 내 | WAS Reload 확인 팝업 열기 | 쓰기 권한 |

### 개별 기능 요구사항

- 속성 상세 팝업에서 해당 속성의 변경 이력을 조회할 수 있어야 한다.
- WAS 인스턴스별 속성 반영 현황을 확인할 수 있어야 한다.
- 유사 속성 검색 기능이 제공되어야 한다 (유사한 이름/값의 속성을 찾아 표시).
- WAS 인스턴스를 선택하여 속성을 반영(Reload)할 수 있어야 한다.
- 속성 그룹 단위로 일괄 관리할 수 있어야 한다.
- WAS별 값 오버라이드: WAS 인스턴스별로 속성값을 다르게 설정할 수 있어야 한다. 오버라이드된 값이 있는 속성은 목록에서 아이콘으로 구분되어야 한다.
- 백업/복원: 속성 전체를 특정 시점으로 백업하고, 필요 시 이전 시점으로 복원할 수 있어야 한다.
- 유사 복사: 기존 속성을 선택하여 새 속성을 등록할 때 값을 복사할 수 있어야 한다.
- 파일 내보내기: 현재 속성 설정을 properties/yaml 파일 형태로 내보낼 수 있어야 한다.

### 비기능 요구사항

| 항목 | 요구사항 |
|------|----------|
| 응답 속도 | 목록 조회 결과가 3초 이내에 표시되어야 한다 |

## XML Property 관리

**화면 목적**: XML 기반 설정 파일을 생성/삭제하고, 파일 내 항목(Key-Value)을 편집하며, WAS 인스턴스에 변경 사항을 반영하는 화면이다.

**접근 경로**: GNB > 인프라 관리 > XML Property 관리

**화면 유형**: 특수 화면 (템플릿 D - 좌우 분할 패널)

### 화면 레이아웃 구성

| 영역 | 설명 |
|------|------|
| 좌측 패널 | 파일 목록 (파일명, 설명, 수정일시) |
| 우측 패널 | 선택된 파일의 항목 목록 (Key-Value 편집 영역) |
| 버튼 영역 | 파일 추가, 항목 저장, Reload 버튼 |

### 목록 표시 (좌측 파일 목록)

| 컬럼명 | 데이터 유형 | 정렬 가능 | 비고 |
|--------|-------------|-----------|------|
| 파일명 | 텍스트 | 불가 | 클릭 시 우측에 항목 표시 |
| 파일 설명 | 텍스트 | 불가 | - |
| 최종 수정일시 | 날짜/시간 | 불가 | - |
| 최종 수정자 | 텍스트 | 불가 | - |

### 파일 등록 팝업

| 항목명 | 입력 유형 | 필수 여부 | 수정 가능 여부 | 유효성 조건 |
|--------|-----------|-----------|----------------|-------------|
| 파일명 | 텍스트 입력 | 필수 | 등록 시만 가능 | 빈 값 불가 |
| 파일 설명 | 텍스트 입력 | 선택 | 가능 | - |

### 버튼 및 액션

| 버튼명 | 노출 조건 | 클릭 동작 | 권한 제어 |
|--------|-----------|-----------|-----------|
| 파일 추가 | 항상 노출 | 파일 등록 팝업 열기 | 쓰기 권한 |
| 파일 삭제 | 파일 선택 시 | 확인 팝업 후 파일 삭제 | 쓰기 권한 |
| 항목 저장 | 파일 선택 시 | 우측 패널 항목 일괄 저장 | 쓰기 권한 |
| Reload | 파일 선택 시 | WAS Reload 확인 팝업 열기 | 쓰기 권한 |
| 전체 Reload | 항상 노출 | 모든 파일을 선택된 WAS에 일괄 반영 | 쓰기 권한 |

### 개별 기능 요구사항

- 좌측 파일 목록에서 파일을 클릭하면 우측에 해당 파일의 항목(Key-Value) 목록이 표시되어야 한다.
- 우측 항목 목록에서 인라인 편집으로 값을 수정할 수 있어야 한다.
- 새 항목을 추가하고 기존 항목을 삭제할 수 있어야 한다.
- Reload 실행 시 WAS 인스턴스를 선택하는 팝업이 표시되어야 하며, 각 인스턴스별 반영 결과(성공/실패)가 표시되어야 한다.
- 전체 Reload 시 모든 파일이 선택된 WAS 인스턴스에 일괄 반영되어야 한다.

### 비기능 요구사항

| 항목 | 요구사항 |
|------|----------|
| 응답 속도 | 파일 목록 및 항목 조회가 3초 이내에 완료되어야 한다 |

## 운영정보 Reload

**화면 목적**: 변경된 시스템 설정 정보를 WAS 인스턴스에 실시간으로 반영(Reload)하는 화면이다.

**접근 경로**: GNB > 인프라 관리 > 운영정보 Reload

**화면 유형**: 특수 화면 (템플릿 D - 체크박스 + 결과)

### 화면 레이아웃 구성

| 영역 | 설명 |
|------|------|
| Reload 대상 선택 영역 | Reload 항목 목록 (체크박스 포함) |
| WAS 인스턴스 선택 영역 | 대상 인스턴스 목록 (체크박스 포함) |
| 실행 버튼 영역 | Reload 실행 버튼 |
| 결과 영역 | Reload 결과 표시 |

### 버튼 및 액션

| 버튼명 | 노출 조건 | 클릭 동작 | 권한 제어 |
|--------|-----------|-----------|-----------|
| Reload 실행 | 항상 노출 | 선택된 항목을 선택된 인스턴스에 Reload | 쓰기 권한 |

### 개별 기능 요구사항

- Reload 대상 항목을 체크박스로 선택할 수 있어야 한다.
- WAS 인스턴스를 체크박스로 선택할 수 있어야 한다.
- 전체 선택/해제 기능이 제공되어야 한다.
- Reload 실행 후 각 인스턴스/항목별 결과(성공/실패/메시지)가 표시되어야 한다.
- Reload 진행 중 로딩 표시가 나타나야 한다.

### 비기능 요구사항

| 항목 | 요구사항 |
|------|----------|
| 응답 속도 | Reload 요청 후 결과가 10초 이내에 표시되어야 한다 |

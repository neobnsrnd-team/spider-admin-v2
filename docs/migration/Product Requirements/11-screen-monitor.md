# [모니터링] 화면 요구사항

---

## 현황판 관리

**화면 목적**: 모니터링 대시보드에 표시할 항목을 등록하고 관리하는 화면이다.

**접근 경로**: GNB > 모니터링 > 현황판 관리

**화면 유형**: 표준 CRUD 화면 (템플릿 A 참조)

### 화면 레이아웃 구성

| 영역 | 설명 |
|------|------|
| 검색 필터 영역 | 상단, 검색 유형 + 검색어 + 조회 건수 + 조회 버튼 |
| 도구 모음 | 목록 상단 우측, 엑셀 / 출력 버튼 |
| 목록 영역 | 중앙, 현황판 항목 목록 테이블 |
| 페이지 네비게이션 | 목록 하단 |
| 버튼 영역 | 목록 하단, 등록 버튼 |

### 검색 / 필터

| 필터 항목명 | 입력 유형 | 필수 여부 | 기본값 | 유효성 조건 |
|-------------|-----------|-----------|--------|-------------|
| 검색 유형 | 드롭다운 (모니터명 / 모니터 ID) | 선택 | 모니터명 | - |
| 검색어 | 텍스트 입력 | 선택 | 빈 값 | - |

### 목록 표시

| 컬럼명 | 데이터 유형 | 정렬 가능 | 비고 |
|--------|-------------|-----------|------|
| 모니터 ID | 텍스트 | 가능 | - |
| 모니터명 | 텍스트 | 가능 | - |
| 모니터 쿼리 | 텍스트 | 가능 | 말줄임 처리 |
| 새로고침 주기 | 숫자 | 가능 | 초 단위 |
| 사용 여부 | 상태 아이콘 | 불가 | Y/N |

- 목록 클릭 시 동작: 모니터 수정 팝업 열림
- 기본 정렬 기준: 모니터 ID 오름차순
- 페이지당 기본 표시 건수: 20건

### 등록 / 수정 팝업

| 항목명 | 입력 유형 | 필수 여부 | 수정 가능 여부 | 유효성 조건 |
|--------|-----------|-----------|----------------|-------------|
| 모니터 ID | 텍스트 입력 | 필수 | 등록 시만 가능 | 영문/숫자, 최대 50자 |
| 모니터명 | 텍스트 입력 | 필수 | 가능 | 최대 100자 |
| 모니터 쿼리 | 텍스트 영역 | 필수 | 가능 | 최대 2000자 |
| 새로고침 주기 | 숫자 입력 | 필수 | 가능 | 1~3600 (초 단위) |
| 표시 순서 | 숫자 입력 | 선택 | 가능 | 0 이상 |
| 사용 여부 | 드롭다운 (Y/N) | 필수 | 가능 | 기본값: Y |
| 설명 | 텍스트 영역 | 선택 | 가능 | 최대 200자 |

### 버튼 및 액션

| 버튼명 | 노출 조건 | 클릭 동작 | 권한 제어 |
|--------|-----------|-----------|-----------|
| 등록 | 항상 노출 | 모니터 등록 팝업 열기 | 쓰기 권한 |
| 엑셀 | 항상 노출 | 검색 결과 엑셀 다운로드 | 전체 |
| 출력 | 항상 노출 | 목록 인쇄 | 전체 |
| 저장 (팝업) | 팝업 열림 시 | 모니터 정보 저장 | 쓰기 권한 |
| 삭제 (팝업) | 수정 모드 시 | 확인 팝업 후 삭제 | 쓰기 권한 |

### 비기능 요구사항

| 항목 | 요구사항 |
|------|----------|
| 응답 속도 | 목록 조회 결과가 3초 이내에 표시되어야 한다 |

## WAS별 Gateway 정보

**화면 목적**: WAS 인스턴스별 게이트웨이 연결 상태를 조회하고, 게이트웨이의 활성/비활성 상태를 확인하는 화면이다.

**접근 경로**: GNB > 모니터링 > WAS별 Gateway 정보

**화면 유형**: 조회 전용 화면 (템플릿 B 참조)

### 화면 레이아웃 구성

| 영역 | 설명 |
|------|------|
| WAS 인스턴스 선택 영역 | 상단, WAS 인스턴스 드롭다운 + 조회 버튼 |
| 게이트웨이 카드 영역 | 중앙, WAS 인스턴스별 게이트웨이 상태를 카드 형태로 표시 |

### 검색 / 필터

| 필터 항목명 | 입력 유형 | 필수 여부 | 기본값 | 유효성 조건 |
|-------------|-----------|-----------|--------|-------------|
| WAS 인스턴스 | 드롭다운 (전체 + 인스턴스 목록) | 선택 | 전체 | API에서 동적 로드 |

### 게이트웨이 카드 표시

각 WAS 인스턴스별로 연결된 게이트웨이 목록이 카드 형태로 표시된다.

| 카드 내 항목 | 데이터 유형 | 비고 |
|-------------|-------------|------|
| WAS 인스턴스 ID | 텍스트 | 카드 제목 |
| WAS IP:PORT | 텍스트 | 카드 부제 |
| 게이트웨이 ID | 텍스트 | 각 게이트웨이 행 |
| 게이트웨이명 | 텍스트 | - |
| 상태 | 상태 배지 | 활성=초록, 비활성=빨강 |
| 연결 시각 | 날짜/시간 | 마지막 연결 시각 |

### 개별 기능 요구사항

- 게이트웨이별 활성/비활성 상태가 색상으로 시각 구분되어야 한다 (활성=초록, 비활성=빨강).
- WAS 인스턴스별로 연결된 게이트웨이 목록이 그룹화되어 표시되어야 한다.
- "전체" 선택 시 모든 WAS 인스턴스의 게이트웨이 정보가 한 화면에 표시되어야 한다.
- 비활성 게이트웨이가 있는 WAS 인스턴스 카드는 시각적으로 강조(빨간색 테두리)되어야 한다.

### 비기능 요구사항

| 항목 | 요구사항 |
|------|----------|
| 응답 속도 | 게이트웨이 정보 조회가 5초 이내에 표시되어야 한다 |

## WAS별 Gateway 모니터링

**화면 목적**: 게이트웨이 상태를 실시간으로 모니터링하고, 상태 변화를 추적하는 대시보드 화면이다.

**접근 경로**: GNB > 모니터링 > WAS별 Gateway 모니터링

**화면 유형**: 모니터링 화면 (템플릿 C 참조)

### 화면 레이아웃 구성

| 영역 | 설명 |
|------|------|
| 제어 영역 | 상단, 새로고침 주기 설정 + 수동 새로고침 버튼 + 자동 갱신 ON/OFF |
| 전체 요약 영역 | 상단 우측, 전체 게이트웨이 중 정상/비정상 건수 요약 |
| 모니터링 대시보드 영역 | 중앙, WAS별 게이트웨이 실시간 상태 표시 |
| 이력 영역 | 하단, 최근 상태 변화 이력 (최대 20건) |

### 제어 영역

| 항목명 | 입력 유형 | 기본값 | 비고 |
|--------|-----------|--------|------|
| 새로고침 주기 | 드롭다운 (5초/10초/30초/60초) | 10초 | - |
| 자동 갱신 | 토글 스위치 | ON | ON/OFF 전환 |
| 수동 새로고침 | 버튼 | - | 즉시 갱신 |

### 모니터링 대시보드

| 항목명 | 데이터 유형 | 비고 |
|--------|-------------|------|
| WAS 인스턴스 ID | 텍스트 | 그룹 헤더 |
| 게이트웨이 ID | 텍스트 | - |
| 현재 상태 | 상태 표시 | 정상=초록 원, 비정상=빨강 깜빡임 |
| 연결 수 | 숫자 | 현재 활성 연결 수 |
| 마지막 응답 시간 | 숫자 | ms 단위 |
| 상태 변화 시각 | 날짜/시간 | 마지막 상태 변경 시각 |

### 상태 변화 이력

| 컬럼명 | 데이터 유형 | 비고 |
|--------|-------------|------|
| 변화 일시 | 날짜/시간 | - |
| WAS 인스턴스 | 텍스트 | - |
| 게이트웨이 | 텍스트 | - |
| 이전 상태 | 상태 배지 | - |
| 변경 상태 | 상태 배지 | - |

- 최근 20건의 상태 변화 이력이 시간 역순으로 표시되어야 한다.

### 개별 기능 요구사항

- 5~10초 간격으로 자동 갱신되어야 한다 (설정 가능).
- 비정상 상태의 게이트웨이가 빨간색 깜빡임으로 시각적 강조되어야 한다.
- 게이트웨이 상태가 변경되면 이력 영역에 자동 추가되어야 한다.
- 자동 갱신 중에도 화면이 깜빡이지 않고 부드럽게 갱신되어야 한다.
- 네트워크 연결이 끊어지면 "연결이 끊어졌습니다. 재연결 중..." 메시지가 표시되어야 한다.

### 비기능 요구사항

| 항목 | 요구사항 |
|------|----------|
| 갱신 주기 | 기본 10초, 최소 5초 ~ 최대 60초 설정 가능 |
| 응답 속도 | 갱신 요청 후 1초 이내에 데이터가 반영되어야 한다 |
| 이력 보존 | 화면 내 최근 20건의 상태 변화 이력이 유지되어야 한다 |

## 성능 모니터링

**화면 목적**: WAS 인스턴스별로 처리 항목(Database, Web, Message, IbatisQuery)의 수행 횟수와 수행 시간을 실시간으로 조회하는 화면이다. 120초마다 자동 갱신된다.

**접근 경로**: GNB > 모니터링 > 성능 모니터링

**화면 유형**: 모니터링 화면 (템플릿 C 참조)

### 화면 레이아웃 구성

| 영역 | 설명 |
|------|------|
| 검색 필터 영역 | 인스턴스 ID 드롭다운 + 카테고리 드롭다운 + 키워드 필터 + 평균수행시간(초) 입력 + 조회 버튼 |
| 자동 갱신 표시 | 타이머 카운트다운 (120초 주기) + 전체 건수 표시 |
| 목록 영역 | 성능 데이터 목록 테이블 |
| 페이지 네비게이션 | 목록 하단 |

### 검색 / 필터

| 필터 항목명 | 입력 유형 | 필수 여부 | 기본값 | 유효성 조건 |
|-------------|-----------|-----------|--------|-------------|
| 인스턴스 ID | 드롭다운 (전체 + 인스턴스 목록) | 선택 | 전체 | - |
| 카테고리 | 드롭다운 (Database / Web / Message / IbatisQuery) | 선택 | 전체 | - |
| 키워드 | 텍스트 입력 | 선택 | 빈 값 | 처리 항목명 필터링 |
| 평균수행시간 | 숫자 입력 | 선택 | 1 | 초 단위, 이 값 이상인 항목만 표시 |

### 목록 표시

| 컬럼명 | 데이터 유형 | 정렬 가능 | 비고 |
|--------|-------------|-----------|------|
| 상세 정보 | 텍스트 | 가능 | 처리 항목 식별 정보 |
| 총 수행횟수 | 숫자 | 가능 | - |
| 총 수행시간 | 숫자 | 가능 | 밀리초 |
| 최대 수행시간 | 숫자 | 가능 | 밀리초 |
| 최소 수행시간 | 숫자 | 가능 | 밀리초 |
| 평균 수행시간 | 숫자 | 가능 | 밀리초 |

- 기본 정렬 기준: 평균 수행시간 내림차순
- 페이지당 기본 표시 건수: 30건

### 버튼 및 액션

| 버튼명 | 노출 조건 | 클릭 동작 | 권한 제어 |
|--------|-----------|-----------|-----------|
| 조회 | 항상 노출 | 현재 조건으로 데이터 새로고침 | 전체 |
| 초기화 | 항상 노출 | 검색 조건 초기화 | 전체 |

### 개별 기능 요구사항

- 화면 진입 시 즉시 조회가 실행되어야 한다.
- 120초마다 자동 갱신되며, 화면에 남은 시간이 카운트다운으로 표시되어야 한다.
- 평균수행시간 필터로 지정한 값 이상인 항목만 표시하여 성능 병목 구간을 빠르게 식별할 수 있어야 한다.

### 비기능 요구사항

| 항목 | 요구사항 |
|------|----------|
| 응답 속도 | 데이터 갱신이 3초 이내에 완료되어야 한다 |

## 모니터링 통계

**화면 목적**: URI 단위의 성능 통계를 일자별/시간별로 조회하고, 상세 통계와 시간대별 분석 데이터를 확인하는 화면이다.

**접근 경로**: GNB > 모니터링 > 모니터링 통계

**화면 유형**: 조회 전용 화면 (템플릿 B 참조)

### 화면 레이아웃 구성

| 영역 | 설명 |
|------|------|
| 검색 필터 영역 | 조회일자 + 로그시간(시간별) + 인스턴스 + URI + 조회 버튼 |
| 목록 영역 | URI별 성능 통계 테이블 |
| 페이지 네비게이션 | 목록 하단 |

### 검색 / 필터

| 필터 항목명 | 입력 유형 | 필수 여부 | 기본값 | 유효성 조건 |
|-------------|-----------|-----------|--------|-------------|
| 조회일자 | 날짜 선택 | 선택 | 오늘 | - |
| 로그시간 | 드롭다운 (시간별) | 선택 | 전체 | - |
| 인스턴스 | 드롭다운 | 선택 | 전체 | - |
| URI | 텍스트 입력 | 선택 | 빈 값 | - |

### 목록 표시

| 컬럼명 | 데이터 유형 | 정렬 가능 | 비고 |
|--------|-------------|-----------|------|
| URI | 텍스트 | 가능 | - |
| 수행횟수 | 숫자 | 가능 | - |
| 평균시간 | 숫자 | 가능 | 밀리초 |
| DB I/O | 숫자 | 가능 | 횟수 |
| DB I/O(시간×회) | 숫자 | 가능 | 시간 × 횟수 |
| LOOKUP | 숫자 | 가능 | 횟수 |
| LOOKUP(시간×회) | 숫자 | 가능 | - |
| ACTION TOTAL | 숫자 | 가능 | - |
| DB R / DB W / MSG | 숫자 | 가능 | 읽기/쓰기/메시지 횟수 |
| BIZ TOTAL | 숫자 | 가능 | - |
| 출력시간 | 숫자 | 가능 | 밀리초 |
| 통계분석 | 버튼 | 불가 | 클릭 시 상세 통계 팝업 |
| 시간별 통계 | 버튼 | 불가 | 클릭 시 시간대별 통계 팝업 |

- 기본 정렬 기준: 수행횟수 내림차순

### 버튼 및 액션

| 버튼명 | 노출 조건 | 클릭 동작 | 권한 제어 |
|--------|-----------|-----------|-----------|
| 조회 | 항상 노출 | 조건으로 통계 조회 | 전체 |
| 통계분석 | 목록 행 내 | 해당 URI의 상세 분석 팝업 | 전체 |

# [인증] 화면 요구사항

---

## 로그인

**화면 목적**: 사용자가 시스템에 접속하기 위해 사용자 ID와 비밀번호를 입력하여 인증하는 화면이다.

**접근 경로**: 시스템 최초 접근 시 자동 이동 / 로그아웃 후 이동

**화면 유형**: 특수 화면 (템플릿 D - 단일 카드)

### 화면 레이아웃 구성

| 영역 | 설명 |
|------|------|
| 배경 | 그라데이션 배경의 전체 화면 |
| 로그인 카드 | 화면 중앙에 최대 너비 420px의 카드 형태 |
| 카드 상단 | 시스템 로고, 시스템명, 환영 메시지 |
| 카드 본문 | 사용자 ID 입력, 비밀번호 입력, 로그인 유지 체크박스, 로그인 버튼 |
| 카드 하단 | 저작권 표시 |

### 입력 항목

| 항목명 | 입력 유형 | 필수 여부 | 수정 가능 여부 | 유효성 조건 |
|--------|-----------|-----------|----------------|-------------|
| 사용자 ID | 텍스트 입력 | 필수 | - | 빈 값 불가 |
| 비밀번호 | 비밀번호 입력 | 필수 | - | 빈 값 불가 |
| 로그인 상태 유지 | 체크박스 | 선택 | - | - |

### 버튼 및 액션

| 버튼명 | 노출 조건 | 클릭 동작 | 권한 제어 |
|--------|-----------|-----------|-----------|
| 로그인 | 항상 노출 | 입력된 자격 증명으로 인증 요청 | 전체 |

### 개별 기능 요구사항

- 비밀번호 입력 필드에 표시/숨김 토글 버튼이 제공되어야 한다 (눈 모양 아이콘).
- Enter 키 입력 시 로그인이 실행되어야 한다.
- 로그인 실패 시 화면 상단에 오류 메시지(빨간색 배경)가 표시되어야 하며, 5초 후 자동으로 사라져야 한다.
- 로그인 성공 시 대시보드(홈) 화면으로 이동되어야 한다.
- "로그인 상태 유지" 선택 시 브라우저 종료 후에도 인증이 유지되어야 한다.
- 이미 로그인된 상태에서 로그인 화면 접근 시 홈으로 자동 이동되어야 한다.

### 비기능 요구사항

| 항목 | 요구사항 |
|------|----------|
| 응답 속도 | 로그인 인증 결과가 2초 이내에 표시되어야 한다 |
| 보안 | 비밀번호는 네트워크 전송 시 암호화되어야 한다 |

## 개인정보수정

**화면 목적**: 로그인한 사용자가 자신의 프로필 정보(이름, 이메일, 비밀번호 등)를 조회하고 수정하는 화면이다.

**접근 경로**: 상단 바 > 사용자 정보 영역 클릭 또는 GNB > 개인정보수정

**화면 유형**: 특수 화면 (템플릿 D - 단일 폼)

### 화면 레이아웃 구성

| 영역 | 설명 |
|------|------|
| 프로필 폼 영역 | 2열 격자 레이아웃의 사용자 정보 입력 폼 |
| 버튼 영역 | 폼 하단에 저장 버튼 |

### 입력 항목

| 항목명 | 입력 유형 | 필수 여부 | 수정 가능 여부 | 유효성 조건 |
|--------|-----------|-----------|----------------|-------------|
| 사용자 ID | 텍스트 (읽기 전용) | - | 불가 | - |
| 사용자명 | 텍스트 입력 | 필수 | 가능 | 최대 50자 |
| 이메일 | 텍스트 입력 | 선택 | 가능 | 이메일 형식, 최대 100자 |
| 전화번호 | 텍스트 입력 | 선택 | 가능 | 최대 13자 |
| 현재 비밀번호 | 비밀번호 입력 | 비밀번호 변경 시 필수 | 가능 | - |
| 새 비밀번호 | 비밀번호 입력 | 선택 | 가능 | 최대 50자 |
| 새 비밀번호 확인 | 비밀번호 입력 | 새 비밀번호 입력 시 필수 | 가능 | 새 비밀번호와 일치 |

### 버튼 및 액션

| 버튼명 | 노출 조건 | 클릭 동작 | 권한 제어 |
|--------|-----------|-----------|-----------|
| 저장 | 항상 노출 | 변경된 프로필 정보 저장 | 본인만 |

### 개별 기능 요구사항

- 비밀번호를 변경하지 않을 경우 비밀번호 필드를 비워두면 기존 비밀번호가 유지되어야 한다.
- 새 비밀번호와 비밀번호 확인이 일치하지 않으면 저장이 불가하여야 한다.

### 비기능 요구사항

| 항목 | 요구사항 |
|------|----------|
| 응답 속도 | 프로필 조회 및 저장이 2초 이내에 완료되어야 한다 |

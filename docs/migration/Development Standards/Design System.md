# Design System

## 1. 개요

프로젝트의 디자인 시스템을 정의한다. **IBM Carbon Design System**을 기반으로 하며, 컴포넌트·레이아웃·색상·타이포그래피 등 시각 체계를 통일한다.

> **IBM Carbon Design System:** https://carbondesignsystem.com

본 문서는 프로젝트 적용 시 준수할 핵심 규칙만 요약한다. 컴포넌트별 상세 가이드라인은 공식 문서를 참고한다.

---

## 2. 레이아웃

### 2.1 8pt 그리드 시스템

모든 간격과 크기를 **8pt 단위**로 설정한다. 미세 조정이 필요하면 4pt 단위를 허용한다.

| 단계 | 값 | 용도 |
|------|-----|------|
| **Compact** | 8pt | 관련 요소 간 밀착 간격 |
| **Regular** | 16pt | 기본 컴포넌트 간 간격 |
| **Generous** | 24pt | 섹션 간 구분 |
| **Spacious** | 32pt 이상 | 주요 영역 간 구분 |

### 2.2 반응형 기준점

| 구간 | 기준 | 레이아웃 전략 |
|------|------|-------------|
| **Mobile** | ~768px 미만 | 사이드바 숨김, 햄버거 메뉴 |
| **Tablet** | 768px~1280px | 사이드바 축소 또는 선택적 표시 |
| **Desktop** | 1280px 이상 | 사이드바(220px) 상시 표시, 다중 컬럼 |

> 이 프로젝트의 주 대상은 **Desktop**이다.

## 3. 색상

### 3.1 시맨틱 컬러

| 토큰 | 용도 |
|------|------|
| **Primary** | 주요 액션, 브랜드 강조 |
| **Secondary** | 보조 액션, 부가 요소 |
| **Destructive** | 삭제, 경고 액션 |
| **Success** | 성공 상태 |
| **Warning** | 주의 상태 |
| **Background** | 화면 배경 |
| **Surface** | 카드·시트 배경 |

하드코딩된 색상값을 직접 사용하지 않는다. `tailwind.config.js`에 정의된 색상 토큰을 사용한다.

### 3.2 대비 기준

WCAG AA 기준(4.5:1) 이상의 대비를 확보한다. 색상만으로 정보를 전달하지 않는다.

---

## 4. 타이포그래피

역할 기반 타이포그래피 체계를 사용한다. 절대 크기보다 **역할(role)** 로 서체를 지정한다.

| 규칙 | 기준 |
|------|------|
| **최소 본문 크기** | 11pt 이하로 내려가지 않는다 |
| **행간(Line Height)** | 글자 크기의 1.2~1.5배 |
| **문단 최대 폭** | 한 줄에 40~60자(한글) 또는 60~80자(영문) |
| **서체 수 제한** | 하나의 화면에 최대 2개 서체 패밀리 |

## 5. 컴포넌트

### 5.1 버튼

| 계층 | 스타일 | 용도 |
|------|--------|------|
| **Primary** | 채워진 배경 + 대비 텍스트 | 화면의 주요 액션 (1개) |
| **Secondary** | 테두리 또는 톤 배경 | 보조 액션 |
| **Tertiary** | 텍스트만 (밑줄 없음) | 부가 액션, 취소 |

### 5.2 데이터 테이블

| 규칙 | 설명 |
|------|------|
| 컬럼 수 제한 | 한 화면에 7±2개 기준 |
| 숫자·날짜 우측 정렬 | 자릿수 비교 용이 |
| 식별자 컬럼 고정 | 가로 스크롤 시 ID·이름 좌측 고정 |

### 5.3 페이지네이션

**페이지 번호** 방식을 사용한다. 더 보기와 무한 스크롤은 사용하지 않는다.

| 규칙 | 설명 |
|------|------|
| 총 건수 표시 | 전체 결과 수를 항상 표시한다 |
| 페이지당 건수 선택 | 10/20/50/100/500 중 선택 (기본값 10) |

### 5.4 모달 / 시트

| 규칙 | 설명 |
|------|------|
| **닫기 수단 명확** | 배경 탭을 통한 닫기를 금지한다. 명시적 버튼으로만 닫는다 |
| **연속 모달 허용** | 모달 위에 모달 호출을 허용한다. 최상위 모달만 노출한다 |
| **상태 보존** | 상위 모달이 열려도 하위 모달의 상태는 유지한다 |
| **순차적 복구** | 최상위 모달이 닫히면 직전 단계 모달이 이전 상태 그대로 표시된다 |

### 5.5 알림 (Alert / Confirm)

Alert과 Confirm 두 가지 다이얼로그만 사용한다. Toast·인라인 배너는 사용하지 않는다.

### 5.6 시스템 상태 패턴

데이터를 표시하는 모든 뷰는 4가지 상태를 처리해야 한다.

| 상태 | 필수 요소 |
|------|----------|
| **Loading** | 스켈레톤 또는 스피너 + 접근성 레이블 |
| **Empty** | 아이콘 + 안내 문구 |
| **Error** | 에러 설명 + 재시도 버튼 |
| **Success** | 데이터 + 메타 정보 |

## 6. 접근성

| 원칙 | 설명 |
|------|------|
| **인식 가능** | 모든 정보를 시각·청각·촉각 중 하나 이상으로 인식 가능 |
| **조작 가능** | 키보드, 스위치 등 다양한 입력 방식으로 사용 가능 |
| **이해 가능** | 콘텐츠와 동작이 예측 가능하고 일관됨 |
| **견고함** | 보조 기술(스크린 리더 등)과 호환 |

| 규칙 | 설명 |
|------|------|
| rem 기반 폰트 크기 | px 고정값 대신 rem 단위를 사용한다 |
| 200% 확대 테스트 | 브라우저 텍스트 200% 확대 시 레이아웃 정상 동작 확인 |
| 포커스 링 유지 | `outline: none`으로 포커스 링을 제거하지 않는다 |
| 모달 포커스 트랩 | 모달 내부에서만 Tab이 순환하도록 트랩 적용 |
| `prefers-reduced-motion` 존중 | 모션 감소 설정 시 애니메이션 최소화 |

## 7. 체크리스트

### 디자인 단계

```
□ 1. 8pt 그리드에 맞춰 간격과 크기가 설정되었는가
□ 2. 시맨틱 컬러 토큰을 사용하고, 하드코딩된 색상값이 없는가
□ 3. 모든 텍스트의 대비 비율이 AA 기준(4.5:1) 이상인가
□ 4. 색상 외 보조 수단(아이콘, 텍스트)으로 정보를 전달하는가
□ 5. 버튼 계층(Primary/Secondary/Tertiary)이 적절히 사용되었는가
□ 6. 데이터 뷰의 4가지 상태(Loading/Empty/Error/Success)가 모두 설계되었는가
```

### 개발 단계

```
□ 7. 모든 입력 필드에 고정 레이블이 있는가 (플레이스홀더로 대체하지 않았는가)
□ 8. 모달 열림 시 포커스 진입, 닫힘 시 포커스 복원이 구현되었는가
□ 9. tailwind.config.js에 정의된 커스텀 색상 토큰만 사용했는가
□ 10. 공통 라이브러리(jQuery 등)가 layout.html에서만 로드되는가
```

### QA 단계

```
□ 11. 브라우저 텍스트 200% 확대 시 레이아웃이 깨지지 않는가
□ 12. Tab 키만으로 모든 인터랙티브 요소에 접근 가능한가
□ 13. 데이터 테이블에서 페이지네이션, 정렬이 정상 동작하는가
```

# Design Principle

## 1. 개요

UI/UX 디자인 원칙을 정의한다. **Apple Human Interface Guidelines (HIG)** 를 기반으로 하며, 웹 브라우저 환경에 맞게 적용한다.

> **Apple Human Interface Guidelines:** https://developer.apple.com/design/human-interface-guidelines

본 문서는 프로젝트에 적용하는 핵심 원칙만 요약한다. 상세 가이드라인은 공식 문서를 참고한다.

---

## 2. 핵심 테마

| 테마 | 설명 |
|------|------|
| **명확성** | 텍스트는 읽기 쉽고, 아이콘은 정확하며, 장식은 기능에 종속된다 |
| **존중** | 콘텐츠가 주인공이다. UI 요소는 콘텐츠를 돕되 경쟁하지 않는다 |
| **깊이** | 시각적 레이어와 모션이 계층 구조와 활력을 전달한다 |

---

## 3. 5대 원칙

| 원칙 | 원문 | 핵심 |
|------|------|------|
| **일관성** | Consistency | 동일한 동작에 동일한 UI를 사용한다 |
| **직접 조작** | Direct Manipulation | 화면 요소를 직접 조작하는 느낌을 제공한다 |
| **피드백** | Feedback | 모든 사용자 행동에 즉각 반응한다 |
| **메타포** | Metaphors | 현실 세계의 경험을 디지털에 반영한다 |
| **사용자 제어** | User Control | 사용자가 항상 주도권을 갖는다 |

---

## 4. 체크리스트

```
□ 1. 디자인 원칙 5가지(일관성, 직접 조작, 피드백, 메타포, 사용자 제어)가 반영되었는가
□ 2. 모든 사용자 액션에 시각적 피드백이 제공되는가
□ 3. 되돌릴 수 없는 동작 전에 확인을 요청하는가
□ 4. 네비게이션 패턴(계층형/플랫/모달)이 콘텐츠 구조에 적합한가
□ 5. 현재 위치가 항상 사용자에게 표시되는가
```

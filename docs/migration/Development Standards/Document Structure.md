# Document Structure

이 폴더의 모든 개발 표준 문서가 따르는 공통 구성을 정의한다.

---

## 1. 파일 제목

```markdown
# [문서명]
```

- 파일명(확장자 제외)과 **일치**시킨다.
- 부제목, 기술 스택 접미사, 블록인용(`>`) 등을 제목 아래에 붙이지 않는다.

| 올바른 예 | 잘못된 예 |
|-----------|-----------|
| `# SQL` | `# SQL - MyBatis Infra Based` |
| `# Authentication & Authorization` | `# Authentication & Authorization Architecture` |
| `# Caching` | `# Caching` + `> Spring Boot + Caffeine 기반 ...` |

---

## 2. 개요 섹션

모든 문서의 첫 번째 섹션은 **`## 1. 개요`** 로 고정한다. 문서의 목적, 적용 범위, 핵심 원칙을 기술한다.

```markdown
## 1. 개요

이 문서의 목적과 핵심 원칙을 기술한다.
```

- `## 1. 원칙`, `## 1. 캐시 설정` 등 도메인 특화 이름을 첫 섹션에 사용하지 않는다.
- 기존 블록인용 부제목(`> ...`)의 내용은 개요 본문으로 통합한다.

---

## 3. 본문 섹션

### 3.1 번호 체계

주요 섹션은 `## N.` 형식, 하위 섹션은 `### N.M` 형식으로 순차 번호를 부여한다.

```markdown
## 2. 주제 A

### 2.1 세부 주제
### 2.2 세부 주제

---

## 3. 주제 B

### 3.1 세부 주제
```

### 3.2 섹션 구분선

주요 섹션(`##`) 사이에 `---`를 삽입하여 시각적으로 구분한다.

### 3.3 콘텐츠별 선택 섹션

아래 섹션은 문서 주제에 따라 **포함 여부를 결정**한다. 모든 문서에 필수는 아니지만, 포함 시 명칭을 통일한다.

| 섹션 | 명칭 | 포함 기준 |
|------|------|-----------|
| 금지 사항 | `## N. 금지 사항` | 위반 시 즉시 반려해야 하는 규칙이 있을 때 |
| 파일 구조 | `## N. 파일 구조` | 프로젝트 디렉터리 구조 안내가 필요할 때 |
| 테스트 | `## N. 테스트` | 테스트 패턴 및 체크리스트를 제공할 때 |
| 부록 | `## 부록 — 이 프로젝트 설정 레퍼런스` | 프로젝트 고유 설정 정보가 있을 때 (번호 없음) |

---

## 4. 마지막 번호 섹션 — `## N. 체크리스트`

문서에 체크리스트를 포함하는 경우, 마지막 번호 섹션으로 배치한다.

```markdown
## N. 체크리스트

- [ ] 항목 1
- [ ] 항목 2
```

- 섹션명은 **`체크리스트`** 로 통일한다.
- `온보딩 체크리스트`, `체크리스트 (XXX 작성 시)` 등 변형을 사용하지 않는다.
- 부록이 있는 경우, 체크리스트 → 부록 순서로 배치한다.

---

## 5. 푸터

모든 문서는 동일한 푸터 형식으로 종료한다.

```markdown
---

*Last updated: YYYY-MM-DD*
```

- **`*Last updated: YYYY-MM-DD*`** 형식을 사용한다 (이탤릭체).
- `*작성일:*`, `*작성일: ... | 상태: 초안*` 등 변형을 사용하지 않는다.

---

## 6. 전체 문서 골격

```markdown
# [문서명]

## 1. 개요

[목적, 적용 범위, 핵심 원칙]

---

## 2. [주제 A]

### 2.1 [세부 주제]
### 2.2 [세부 주제]

---

## 3. [주제 B]

...

---

## N. 체크리스트

- [ ] 항목 1
- [ ] 항목 2

---

## 부록 — 이 프로젝트 설정 레퍼런스 (선택)

[프로젝트 고유 설정]

---

*Last updated: YYYY-MM-DD*
```

## 7. 요약

| 항목 | 규칙 |
|------|------|
| 제목 | `# [파일명]` — 접미사·블록인용 없음 |
| 첫 섹션 | `## 1. 개요` 고정 |
| 번호 체계 | `## N.` / `### N.M` 순차 번호 |
| 섹션 구분 | `---` 삽입 |
| 체크리스트 | `## N. 체크리스트` — 마지막 번호 섹션 |
| 부록 | 번호 없이 체크리스트 뒤에 배치 (선택) |
| 푸터 | `*Last updated: YYYY-MM-DD*` |
